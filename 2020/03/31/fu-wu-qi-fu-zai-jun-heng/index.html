服务器负载均衡

SLB

  • 服务器负载均衡就是将本应由一个服务器处理的业务分发给多个服务器来处理，以此提高处理业务的效率和能力。

  • 处理业务的服务器组成服务器集群，对外体现为一台逻辑上的服务器。对于用户来说，访问的是这台逻辑上的服务器，而不知道实际处理业务的是其他服务器，由FW决定如何分配流量给各个服务器。

负载均衡算法

  • 负载均衡算法决定了FW如何分配业务流量到服务器上

FW支持的负载均衡算法如下：

1.简单轮询算法

优点：实现简单，效率较高
缺点：不会考虑每个服务器上的实际负载
简单轮询算法是一个静态算法
适用场景为服务器的性能相近，服务类型比较简单，且每条流对服务器造成的业务负载大致相等

2.加权轮询算法

加权轮询算法是将客户端的业务流量按照一定权重比依次分配到各个服务器上
根据服务器的不同处理能力，给每个服务器分配不同的权值，使其接受相应权值数的服务请求

3.最小连接算法

最小连接算法是将客户端的业务流量分配到并发连接数最小的服务器上。(并发连接数即服务器对应的实时连接数，会话新建或老化都会影响并发连接数的大小)
最小连接算法是动态算法
适用场景为服务器的性能相近，每条流对服务器造成的业务负载大致相等，但是每条流的会话存活时间不同。

4.加权最小连接算法

加权最小连接算法是将客户端的业务流量分配到加权并发连接数最小的服务器上。(加权并发连接数是考虑服务器权重后得到的实时连接数)
适用场景为服务器的性能不同，每一条流对服务器造成的业务负载大致相等，但是每一条流的会话存活时间不同。

5.源IP hash算法

源IP Hash算法是将客户端的IP地址进行Hash运算，得到Hash Index值。FW根据Hash Index值与Hash列表中服务器的对应关系，分配流量至服务器。
适用场景为各个服务器性能差别不大

6.加权源IP hash算法

加权源IP Hash算法是将客户端的IP地址进行Hash运算，得到Hash Index值，而后通过FW调整各个服务器的权重得到Hash表中服务器新的权重值。
适用场景为各个服务器的性能有一定的差异

配置

配置步骤

一、在FW1上的配置

1.开启SLB

slb enable

2.配置SLB

slb //SLB视图下

group 1 fwq //实服务器组
metric roundrobin //算法为简单轮询
health-check type icmp //健康检查
rserver 0 rip 2.1.1.1 port 21
rserver 1 rip 2.1.1.2 port 21
rserver 2 rip 2.1.1.3 port 21

vserver 0 vs //虚拟服务器组
vip 0 1.1.1.100 //虚拟IP
protocol tcp
vport 21
group fwq //关联实服务器组

验证配置
