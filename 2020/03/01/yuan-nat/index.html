源NAT

  • 源NAT是指对报文中的源地址进行转换。
    通过源NAT技术将私网IP地址转换成公网IP地址，使私网用户可以利用公网地址访问Internet。

NAT No-PAT

  • NAT No-PAT是一种NAT转换时只转换地址，不转换端口，实现私网地址到公网地址一对一的地址转换方式。

    如果地址池中的地址已经全部分配出去，则剩余内网主机访问外网时不会进行NAT转换，直到地址池中有空闲地址时才会进行NAT转换

FW上会生成两个Server-map表项：

正向Server-map表项保证特定私网用户访问Internet时，快速转换地址，提高了FW处理效率。
反向Server-map表项允许Internet上的用户主动访问私网用户，将报文进行地址转换。

NAT NO-PAT有两种:

本地（Local）NO-PAT
本地NO-PAT生成的Server-Map表中包含安全区域参数，只有此安全区域的Server可以访问内网Host。
全局（Global）NO-PAT
全局NO-PAT生成的Server-Map表中不包含安全区域参数，一旦建立，所有安全区域的Server都可以访问内网Host。

  • 适用场景：上网用户较少且公网地址数与同时上网的用户数量相同。

NAPT

  • NAPT是一种转换时同时转换地址和端口，实现多个私网地址共用一个或多个公网地址的地址转换方式。

    由于地址转换的同时还进行端口的转换，可以实现多个私网用户共同使用一个公网IP地址上网，FW根据端口区分不同用户，所以可以支持同时上网的用户数量更多。

NAPT方式不会生成Server-map表

  • 适用场景：公网地址数量少，需要上网的私网用户数量大。

Smart NAT

  • Smart NAT是No-PAT方式的一种补充

    Smart NAT是一种可以在No-PAT的NAT模式下，指定某个IP地址预留做NAPT方式的地址转换方式。

Smart NAT的产生：
使用No-PAT方式时，进行地址池的一对一转换。随着内部用户数量的不断增加，地址池中的地址数可能不再能满足用户上网需求，部分用户将得不到转换地址而无法访问Internet。

  • 由于地址转换的同时还进行端口的转换（先是No-PAT的地址用尽后才会使用预留地址进行NAPT的转换），可以实现多个私网用户共同使用一个公网IP地址上网，FW根据端口区分不同用户，所以可以支持同时上网的用户数量更多。

NAPT方式不会生成Server-map表，No-PAT会生成Server-map表

  • 适用场景：平时上网的用户数量少，公网IP地址数量与同时上网用户数基本相同，但个别时段上网用户数激增。

Easy IP

  • Easy IP是一种利用出接口的公网IP地址作为NAT转后的地址，同时转换地址和端口的地址转换方式。

    • 接口IP是动态获取的场景下也可使用
    • 支持端口的转换

Easy IP的产生：使用No-PAT或NAPT方式时均需要多个静态的公网IP地址，且需要配置地址池，会造成大量IP地址的浪费。当公网IP地址为动态获取时，NAT的地址池很难得到正确的配置。

  • 适用场景：当接口的公网IP是动态获取的状态下，只想使用这一个公网IP进行上网，并且私网用户数量多。

  • Easy IP不会产生Server-map

三元组NAT

  • 三元组NAT是一种转换时同时转换地址和端口，实现多个私网地址共用一个或多个公网地址的地址转换方式且允许Internet上的用户主动访问私网用户。

    • 三元组NAT的端口不能复用，保证了内部PC对外呈现的端口的一致性，不会动态变化，但是公网地址利用率低。
    • 支持外部设备通过转换后的地址和端口主动访问内部PC。FW即使没有配置相应的安全策略，也允许此类访问报文通过。

三元组NAT的产生：当内网PC访问Internet时，如果FW采用五元组NAT（NAPT）方式进行地址转换，外部设备无法通过转换后的地址和端口主动访问内部PC。

  • 缺省情况下，端点无关过滤功能处于开启状态。

开启时公网用户访问私网用户时匹配目的Server-map表，不进行安全策略处理。
关闭时匹配安全策略规则，由安全策略规则决定是否转发报文。

  • 三元组NAT可产生正向与反向的Server-map。

  • 三元组NAT有两种：

    • 本地（Local）三元组NAT
    • 全局（Global）三元组NAT

  • FW支持Smart三元组NAT功能，可以根据报文的目的端口来选择分配端口的模式，在一定程度上提高公网地址的利用率。当报文的目的端口属于设置的端口范围之内，就采用NAPT模式来分配端口，如果报文的目的端口不属于设置的端口范围之内，则采用三元组NAT模式来分配端口。
