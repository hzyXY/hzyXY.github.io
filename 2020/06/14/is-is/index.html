IS-IS

  • IS-IS特点：仅用于CLNS环境下。
  • 集成IS-IS特点：支持CLNP和IP双重网络。
  • ISIS协议和IP协议是解耦的，无关联的独立的网络层协议，通过802.3以太网帧进行封装。

引入

TCP/IP网络模式 路由器 终端

OSI7层模型 中间系统（IS） 终端（End System）

由于CLNP没有被流行，所以ISIS在早期发展比较缓慢，比较沉寂。

后来由于ISIS比较灵活，它自身是一种工作在链路层之上的网络层协议，扩展性较强。

  • CLNP（无连接的网络层协议）是OSI模型中的网络层协议，ISIS最初是为了支持CLNP而被开发。

  • ISIS OSI7层模型 用于支持CLNP的网络环境，OSPF TCP/IP 用于IP的网络环境

ISIS工程小组后续对ISIS协议进行扩展，让它能支持IPv4、IPv6。CLNP 所以现在ISIS协议被称为集成ISIS协议。

由于扩展性强，收敛快，工作机制简单，被ISP大量的部署。

IS-IS的基本概念

IS-IS协议区域

  • ISIS协议的骨干区域和非骨干区域路由器的邻居级别来确定
  • ISIS定义路由器级别有3种：level-1、level-2、level-1-2。华为思科路由器默认为L1/2的路由器
  • ISIS邻居级别有两种：L1的邻居、L2的邻居。骨干区域就是由L2的邻居组成的逻辑区域。ISIS非骨干区域就是由L1的邻居组成的逻辑区域。骨干和非骨干的边界在L1/2的路由器上。

IS-IS邻居关系

L1的邻居：建立的前提 区域号一致

L2的邻居：不要求区域号一致

L1的路由器只能建立L1的邻居，L2的路由器只能建立L2的邻居，L1/2的路由器和同区域的L1路由器建立L1邻居，和L2的路由器建立L2邻居。和L1/2的同区域路由器既建立L1的邻居又建立L2的邻居。和L1/2的不同区域的路由器只能建立L2的邻居。

形成邻居关系的条件

IS-IS形成邻居关系的条件

  1. 同一（层次）级别

  2. L1的邻居区域号要一致

  3. L1、L2邻居互联地址必须在同一个网段

  4. L1、L2邻居网络类型必须一致

邻居关系的建立

广播型链路通过IS NET TLV实现3次握手建立邻居。

P2P的链路通过P2P ADJ Stata TLV实现3次握手建立邻居。

P2P的链路上还可以通过2次握手建立邻居关系。

总结：广播型链路仅支持3次握手建立邻居，P2P链路支持3次握手，也可以支持2次握手。

如何判断p2p链路上使用的是3次握手？
查看hello包中是否有P2P ADJ Stata TLV，如有则是3次握手。

命令

isis ppp-negotiation 3-way p2p链路上启用3次握手机制，但不要求接收的hello包携带p2p adj state tlv。

isis ppp-negotiation 3-way only p2p链路上启用3次握手机制，不兼容2次握手。

isis ppp-negotiation 2-way p2p链路上只采用2次握手建立邻居，忽略不处理p2p adj state

ISIS邻居状态

down 没有收到邻居的hello

initialized 收到邻居的hello包，但从中没发现自身的mac地址。也称为one-way

up 收到邻居的hello包，并从中发现有自己的接口mac地址。

ISIS的网络类型

  • OSPF协议默认情况下根据接口的链路层封装协议来确定网络类型。
  • ISIS类似，如果是以太网接口则默认认为是广播型，如果过是PPP，HDLC则默认是P2P，不支持NBMA，P2MP。

IS-IS支持的网络类型

  • 点对点网络，对应PPP、HDLC等链路
  • 广播网络，对应Ethernet等链路

广播链路：DIS和伪节点

  • DIS：Designated Intermediate System 指定中间系统
  • 功能：在广播网络实现更高效的数据库同步

IS-IS地址结构

  • IP协议：使用IP地址对接口进行编址。
  • CLNP协议使用NSAP（网络服务访问点）地址对设备进行编址。

NSAP是OSI协议中用于定位资源的地址

  • 相当于OSI的网络层协议CLNP的地址（类似IP地址的概念）

NET是一类特殊的NSAP（SEL=00）

  • 在路由器上配置IS-IS时，只需要考虑NET即可。

NET（网络实体标识）地址是一种特殊的NSAP地址，即SEL=0的NSAP地址，在ISIS协议由NET地址标识一台路由器
SEL：网络服务鉴别器，1个字节长度，类似于IP报文中的协议号，用于区分上层协议类型。

IS-IS报文类型

广播型链路上所有L1的ISIS报文使用的DMAC地址为
DMAC地址 01:80:C2:00:00:14

广播型链路上所有L2的ISIS报文使用的DMAC地址为
DMAC地址 01:80:C2:00:00:15

P2P型链路上所有级别的ISIS报文使用的DMAC地址为
DMAC地址：09:00:2b:00:00:05

ISIS协议是基于TLV机制来协商各项属性的。
T：属性类型
L：长度
V：取值

TLV机制运行路由协议只携带需要的属性，不需要属性可以不携带，扩展性强，如果过协议需要支持新的特性，则只需要开发新TLV属性即可支持。

OSPFv2不是基于TLV机制路由协议，报文字段都是固定的，所以扩展性比较差。

HELLO PDU

HELLO PDU（IIH）用于建立和维持邻居关系

  • Level-1 LAN IIH：用于在广播型链路上建立L1的邻居
  • Level-2 LAN IIH：用于在广播型链路上建立L2的邻居
  • P2P IIH：用于在P2P链路上建立ISIS的邻居

作用：建立和维护邻居关系

问题：为什么要在广播型链路上设计两种hello报文，而P2P链路只有一种hello报文？
答：因为广播型链路上存在多路访问的链路，这条链路上可存在多台路由器。邻居关系又分为级别一、级别二。某些路由器只需要建立级别一的邻居关系，它们只需要发送级别一的hello包维护。另一些路由器需要建立级别二的邻居关系，它们就只需要发送级别2的hello包维护。不同级别的邻居设计出不同级别的hello包分别去进行维护，这样会更加细致一些，控制起来也更方便一些，可以独立进行维护。而点到点链路只有两台路由器，不存在多种不同级别邻居的可能性。要么是级别1，要么是级别2，要么既是级别1、也是级别2。所以只需要一种就可以搞定。

默认情况下 L1/2的路由器在广播型链路上既发L1的hello，也发L2的hello报文。

L1/2的路由器在广播型链路上发送hello报文的级别与接口的级别相关，和接口级别取交集。

L1的路由器在广播型链路上只发L1的hello，同理L2的路由器在广播型链路上只发L2的hello报文。跟链路级别无关。

hello报文circuit type字段取值特点：

  • 路由器类型level-1，则取值为0x01，路由器类型为level-2，则取值为0x02，与链路级别无关。
  • 如果路由器类型为level1-2，则与链路级别取交集，链路为L1时，则取值0x01，链路为L2时，则取值0x02
  • 如果取值为0x03则路由器为level-12的路由器。并且链路为L1-2。

邻居失效时间（Holding timer：告知邻居本端失效时间。）：hello报文发送间隔的3倍。

isis.hello.priority 用于DIS的选举，取值范围0-127，越大越优先，0也可以参与DIS的选举。如果过优先级相同，则比较接口的MAC地址，越大越优先。默认为64。

IS Neighbor t=6，l=6 //邻居接口的MAC地址
IP Interface address t=132，l=4 //自身接口的IP地址
Protocols Supported t=129，l=1 //所支持的协议

命令修改

isis timer hello 4 level-1 修改hello间隔

isis timer holding-multiplier 4 level-1 告知邻居本端失效时间

LSP PDU

LSP PDU用于交换链路状态信息

  • Level-1 LSP：描述非骨干区域的链路状态，对应于OSPF协议非骨干区域中的1类或2类LSA
  • Level-2 LSP：描述骨干区域的链路状态，对应于OSPF协议骨干区域中的1类或2类LSA

作用：用于描述携带LSA（链路状态）信息 。

从报文角度去看LSP分为两种L1 LSP、L2 LSP

从LSP的用途去看LSP分为两种 实节点LSP，伪节点LSP

实节点LSP相当于OSPF的1类LSA，每台ISIS路由器都会产生用于描述自身直连链路状态。

伪节点LSP相当于OSPF的2类LSA，广播型链路上由DIS产生，用于描述广播型链路。

L1路由器只维护L1 LSP
L2路由器只维护L2 LSP
L1/2路由器维护L1 LSP也维护L2 LSP

ISIS协议如何区分不同的LSP呢？
ISIS协议定义通过LSP ID唯一的区分一条LSP

0000.0000.0004.00-00 LSP-ID 8个字节长度

由3个部分组成
LSP ID = system id（6字节）. 伪节点标识符（1字符）- 分片标识符（1字节）

system id标识产生LSP的路由器的system id。类似ospf的adv router

伪节点标识符 取值 0 或者 非0，如果等于0，则代表该LSP是实节点LSP。如果非0，则代表该LSP是伪节点LSP，此时system id为DIS的system id。 类似LSA TYPE。

分片标识符：用于描述同一台路由器产生的不同的LSP

LSP新旧判断机制

  1. 比较LSP SEQ，越大越新
  2. seq相同则判断remaining lifetime（hold time）是否等于0，如果等于0，则认为是最新的LSP，用于删除一条LSP（用于发给对端邻居作删除的动作）。默认为1200s，每隔15分钟，即900s周期性泛洪，更新LSP，同时seq+1。（ISIS是倒计时。与OSPF相反。）
  3. 如果remaining lifetime都不等于0，则比较checksum，越大越新。
  4. 如果checksum一样，则认为是相同的LSP。初始值从0x0000001开始,最大值为0xFFFFFFFF。

SNP PDU
  • 序列号PDU

SNP PDU用于维护LSDB的完整与同步，且为摘要信息

  • CSNP（完全序列号PDU，类似OSPF的DD报文）
    • Level 1 CSNP
    • Level 2 CSNP
  • PSNP（部分序列号PDU，类似于OSPF的LSR或LSACK。）
    • Level 1 PSNP
    • Level 2 PSNP

PSNP报文如何判断是请求还是确认？
PSNP报文中携带的LSP ID的序列号非0的都是用作确认用的，0的均是请求。

PSNP，CSNP携带的LSP的携带信息包括LSP ID，remaning lifetimer，seq，checksum。

DIS

作用：起到维护链路上拓扑数据库一致的作用。

DIS选举

  1. 接口优先级越大越优。

  2. 优先级相同，则比较mac地址，越大越优先。

  3. 默认优先级64。

  4. DIS可以抢夺。

  5. DIS分级别，L1的邻居选举L1 DIS，L2的邻居选举L2 DIS。

  6. DIS优先级等于0的路由器也可以进行DIS的选举。

  7. 没有备份的DIS。

P2P网络LSP交互过程

  • 有确认报文，重传时间为5s。

  • P2P链路无需周期性发送CSNP报文。

广播网络LSP交互过程

  • 无确认报文

  • DIS每10s周期发送CSNP，维护此广播型网络上路由器的LSDB同步。

IS-IS路由过载

OL：过载比特位，通过set-overload命令设置。

其他路由器不会在进行SPF算法时不在沿着OL=1的路由器继续往下计算SPF树，OL的路由器做为树上的末端节点。

L1/2设置OL bit后，将不再执行泄露功能，无论是否配置了路由泄露。

路由过载的功能：
RIP 接口上metric out 将发送的路由改成16跳
EIGRP stub router（末节路由器）
OSPF stub router（末节路由器）将自己直连的链路cost值设置成65535

命令

set-overload [allow {interlevel | external} * ]

external引入到ISIS协议中的外部路由，在引入外部路由的ASBR上配置。

set-overload allow interlevel L12，L1，L2继续描述自身引入的外部路由，对非自身产生的外部路由无效。通常在L1/2路由器上实施。

set-overload allow external L12路由器过载后继续描述AS内部泄露路由以及其他路由器引入的外部路由。通常在ASBR路由器上实施。

set-overload

L12路由器设置OL=1后，默认不把L1路由在L2的LSP进行描述，也不会把L2的泄露的路由在L1的LSP中进行描述，同时ATT=0，也不会描述自身引入的外部路由。

L1，L2路由器设置OL=1后，不描述自身引入的外部路由。

set-overload on-startup 设备重启后的一段时间内保持LSP的OL位置1。过了这段时间就恢复。

set-overload on-startup wait-for-bgp 100 无论BGP邻居是否建立，总是在启动后100s内进行OL置位。

路由计算与交互

L1的路由器只通过L1的LSP描述直连链路信息，跟链路级别无关。
L2的路由器只通过L2的LSP描述直连链路信息，跟链路级别无关。

L1/2的路由器通过L1的LSP也通过L2的LSP描述自身L1/2的直连链路。
L1/2的路由器只通过L2的LSP的LSP描述自身L2的直连链路。
L1/2的路由器通过L1的LSP也通过L2的LSP描述自身L1的直连链路。

骨干区域如何访问非骨干区域？
L1/2的路由器将L1的路由作为自身直连的叶子进行描述，并通过L2的LSP携带再骨干区域泛洪。

非骨干区域如何访问骨干区域呢？
默认情况下L1/2的路由器不会将L2的路由转化成L1的LSP在非骨干区域泛洪，那么ISIS非骨干区域天生类似于OSPF的完全末节区域，通过缺省路由访问访问骨干区域。

OSPF的完全末节区域如何访问其他区域？
通过完全末节区域的ABR产生的3类LSA的缺省路由。

ISIS非骨干区域的缺省路由是怎么计算出来的呢？
由L1/2的路由器产生的L1的LSP中通过ATT bit位，来进行缺省路由的计算。

ATT（骨干区域连接符），用来表明该L1/2的路由器和骨干区域连接着，ATT此时置1。
当L1/2的路由器至少存在一个不在相同区域的L2的邻居时，ATT才会置1。且ATT比特位只会在L1的LSP中置1。

缺省路由的计算

  1. L1的路由器通过ATT置1的LSP得知本区域的L1/2路由器的存在。
  2. 由于L1的路由器和L1/2路由器在同一个区域，所以L1的路由器将自动在路由表中生成一条指向L1/2路由器的缺省路由。
  3. 该路由的下一跳为到L1/2路由器SPF树的下一跳
  4. 该路由的开销为到L1/2路由器的SPF树的开销。

OSPF ABR在自己产生的1类LSA中，通过一个flags来表明自身是ABR，该flags称为B flags。

同一个区域如果过存在多个L1/2的路由器，彼此不进行缺省路由的计算。防止路由环路。

Distribution（DU bit）置1为泄露到非骨干区域的L2路由

命令

ATT bit的控制
attaached-bit advertise always/never 设置ATT bit位总是置1/0。

attached-bit advertise avoid-learning 设置L1的路由器不进行缺省路由的计算。

import-route isis level-2 into level-1 filter-policy ip-prefix 1
L2的路由向L1区域泄露时做路由过滤

路由过滤：在L1/2路由器中进行从L2/L1渗透进L1/L2时，对直连的路由不生效。

ISIS防环

同一个区域存在多台L1/2的路由器时如何实现防环的？即ISSI区域间防环机制

  1. 为什么泄露到非骨干区域的路由DU会置1，而从非骨干区域传到骨干区域的路由DU不会置1呢？
    问题1原因：由于L1的路由默认会转化L2的路由在骨干区域泛洪，为了防止L2的路由泄露到L1的区域后，再次传回骨干区域，将该路由的DU bit置1。L1/2路由器不会将DU=1的路由再次传回骨干区域。（区域间的水平分割）
    问题2原因：L2的路由默认不会传到L1的区域，所以不存在环路风险。

  2. 如果管理员将L2的路由泄露到L1的区域，那么这些路由会再次传回L1的区域吗？
    不会，因为L1的路由优于L2的路由，这样L2的路由就无法再次泄露到非骨干区域。

  3. 为了防止L1/2的路由器优先选择DU置1的路由访问骨干区域，产生次优路径，所以ISIS在这种情况下L2的路由优先DU置1的L1的路由。即L1 > L2 > L1 DU。

面试题

  1. 路由协议工作在那一层？

RIP UDP520，BGP TCP179 工作在应用层

OSPF IP89 工作在网络层之上和传输层在同一层

ISIS 工作在数据链路层之上独立的网络层协议

  1. NET地址由什么组成？

3部分组成，area id，system id，sel。其中area id可变长度至少一个字节，最大13个字节。system id固定6个字节。sel始终为0，一个字节。

  1. 基于TLV机制的协议

EIGRP、BGP、LDP

  1. ospf和isis的区别

OSPF在LSDB同步过程中定义状态机 exstart，exchange，loading，full

ISIS在LSDB同步过程中没有定义任何状态机

ISIS协议如何区分不同的LSP呢？
ISIS协议定义通过LSP ID唯一的区分一条LSP

  1. OSPF的DR，BDR为什么不抢夺？而ISIS为什么可以抢夺？
    主要原因式广播型链路的邻居状态不一致。如果DR能够抢夺，将导致邻居状态发生大面积的变化，影响设备很多。ISIS抢夺，因为所有路由器都维持UP状态，不会引起邻居关系发生变化。稳定性高。但是伪节点LSP会发生变化，对SPF算法有一定的影响。稳定网络中可以忽略该问题。

  2. OSPF的广播链路的网络号是根据几类LSA计算出来的？2类怎么算？
    2类 ls id DR 接口的地址/携带DR接口的掩码。

ISIS的 伪节点LSP是纯的拓扑信息，不携带路由信息。ISIS通过实节点LSP描述P2P的直连路由，也描述广播型链路的直连路由。

  1. 为什么OSPF要要求广播型链路掩码要一致？
    因为ospf是DR用2类LSA描述广播型网络的，且掩码信息为自身地址的掩码。在这个广播型网络中如果存在不同掩码，那么路由会按最长匹配原则来转发路由信息。导致不通。（这是在假设它们可以建立邻居和交互路由信息的前提下。）

  2. 通过缺省路由访问骨干区域存在什么问题？

    缺点：

    1. 次优路径

      如何防止次优路径？

      1. 手动调整缺省路由的开销，通过修改链路开销的实现。
      2. 通过路由泄露，将骨干区域的路由泄露到非骨干区域，泄露到非骨干区域的L2的路由会进行DU bit置1，表示该路由来自骨干区域，是泄露进非骨干区域。

    2. 无法感知明细路由的状态，存在或者失效都通过缺省路由转发业务，导致带宽存在浪费。

    优点：

    1. 收敛快，无法维护明细路由。
    2. 节省设备资源
    3. 增强网络的稳定性

  3. 如何删除LSP里面的路由？
    产生一条新的LSP（新的序列号+1），覆盖掉以前旧（序列号）的LSP。里面不携带不要的路由发送的邻居。

  4. 如何删除一条LSP？
    发送这条LSP的holdtime等于0。

P bit位

代表分区修复功能，即表示该路由器是否支持ISIS的虚链路。

骨干区域修复标志位。用于做虚链路的。但目前各大厂家还未实现，所以置0。

解释：ISIS不是TCP/IP环境下的网络层协议，那么在基于IP的网络中无法实现跨链路建立ISIS邻居的功能。故无法为IP网络支持虚链路功能。
