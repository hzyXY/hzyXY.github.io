1.Layer-2

1.1链路聚合

1.S1和S2之间配置链路聚合，使用手动负载分担模式，基于源目MAC地址负载分担。（2分）

解法：分别在S1和S2上配置Eth-trunk。

interface Eth-Trunk12
mode manual load-balance
load-balance src-dst-mac
trunkport GigabitEthernet 0/0/23
trunkport GigabitEthernet 0/0/24

1.2Link-type

1.在S1，S2，S3，S4互联接口的链路类型为Trunk，允许除了VLAN1外的所有VLAN通过（3分）

在S1，S2，S3，S4上创建vlan 10和vlan 20，配置交换机之间的链路为Trunk，并放行除VLAN1之外的VLAN通过。

vlan batch 10 20
interface GigabitEthernet 0/0/1（相应接口）
port link-type trunk
port trunk allow-pass vlan all
undo port trunk allow-pass vlan 1

2.CE1、CE2的VRRP虚拟IP地址10.3.1.254，为PC1的网关。CE1会周期性发送Sender IP为10.3.1.254，源MAC为00-00-5E-00-01-01的免费ARP。PC1与网关之间的数据包封装在VLAN10中（PC1收发untag的帧）。

在CE1的G0/0/2.10和G0/0/2.20接口上配置VRRP协议，接口地址已经预配。
interface GigabitEthernet0/0/2.10
vrrp vrid 1 virtual-ip 10.3.1.254
vrrp vrid 1 priority 120
vrrp vrid 1 preempt-mode timer delay 60
arp broadcast enable

interface GigabitEthernet0/0/2.20
vrrp vrid 2 virtual-ip 10.3.2.254
arp broadcast enable

3.CE1、CE2的VRRP虚拟IP地址10.3.2.254，为Server1的网关。CE2会周期发送Sender IP为10.3.2.254，源MAC为00-00-5E-00-01-02的免费ARP。Server1与网关之间的数据包封装在VLAN20中（Server收发untag的帧）。

在CE2的G0/0/2.10和G0/0/2.20接口上配置VRRP协议，接口地址已经预配。
interface GigabitEthernet0/0/2.10
vrrp vrid 1 virtual-ip 10.3.1.254
arp broadcast enable

interface GigabitEthernet0/0/2.20
vrrp vrid 2 virtual-ip 10.3.2.254
vrrp vrid 2 priority 120
vrrp vrid 2 preempt-mode timer delay 60
arp broadcast enable

4.VRRP的master设备重启时，在G0/0/2变为UP一分钟后，才能重新成为master。

vrrp vrid 1 preempt-mode timer delay 60
vrrp vrid 2 preempt-mode timer delay 60

1.3MSTP

1.S1、S2、S3、S4都运行MSTP。VLAN10在Instance 10，S1作为Primary Root。VLAN20在Instance 20，S2作为Primary Root。MSTP的region name是HUAWEI，Revision-level为12。

分别在S1,S2,S3,S4上配置MSTP，配置如下：
stp region-configuration
region-name HUAWEI
revision-level 12
instance 10 vlan 10
instance 20 vlan 20
active region-configuration

在S1上配置Instance 10和Instance 20的根桥和备份根桥。
stp instance 10 root primary
stp instance 20 root secondary
在S2上配置Instance 20和Instance 10的根桥和备份根桥。
stp instance 10 root secondary
stp instance 20 root primary

2.除了交换机互联的接口，其他接口要确保不能参与MSTP计算，由Disabled直接转到Forwarding状态。

[LSW1]stp edged-port default
[LSW2]stp edged-port default
[LSW3]stp edged-port default
[LSW4]stp edged-port default
分别在S1和S2的Trunk接口配置非边缘端口：（再把交换机互联的端口设置为非边缘端口）
Interface Eth-Trunk 12
stp edged-port disable
interface GigabitEthernet 0/0/1
stp edged-port disable
interface GigabitEthernet 0/0/12
stp edged-port disable
分别在S3和S4的Trunk接口配置非边缘端口：
interface GigabitEthernet 0/0/1
stp edged-port disable
interface GigabitEthernet 0/0/2
stp edged-port disable

route-policy import permit node 10
if-match ip-prefix 1

ospf 1
import-route direct type 1 route-policy import

2.3ISIS

1.AS 100内Loopback0和互联接口全部开启ISIS协议，其中PE1、PE2路由类型为Level-1，区域号为49.0001；RR1、P1路由类型Level-1-2，区域号为49.0001；ASBR1、ASBR2路由类型为Level-2，区域号为49.0002。各单元的System-ID唯一，cost-style为wide；cost值如图配置（除PE1-RR1之间的逻辑接口外，已预配置）。（1分）

分别在PE1和RR1之间的逻辑接口启用ISIS协议，并设置链路开销。
Interface IP-Trunk 1
isis enable 1
isis cost 1500

2.RR2-P2的ISIS链路类型为P2P。（1分）

Interface GigabitEthernet0/0/0
isis circuit-type p2p

  • 为了保证后续的MPLS VPN中AS 100公网LDP的可达，在RR1和P1上做172.16.0.0/16主机路由Level-2向Level-1路由的渗透。

    RR1上配置路由渗透：
    ip ip-prefix 1 index 10 permit 172.16.0.0 16 greater-equal 32 less-equal 32
    isis 1
    import-route isis level-2 into level-1 filter-policy ip-prefix 1

    P1上配置路由渗透：
    ip ip-prefix 1 index 10 permit 172.16.0.0 16 greater-equal 32 less-equal 32
    isis 1
    import-route isis level-2 into level-1 filter-policy ip-prefix 1

3.在RR2、P2上，ISIS和OSPF双向引入前缀为172.16.0.0/16的主机路由。被引入的协议的cost要继承到引入后的协议中，P2和PE4的Loopback0互访走最优路径。配置要求有最好的扩展性。（8分）

ospf 1
default cost inherit-metric

在RR2上将OSPF的路由引入到ISIS协议，添加Tag为100，并拒绝Tag为200的OSPF路由，即从P2上引入的OSPF路由。
ip ip-prefix 32 index 10 permit 172.16.0.0 16 greater-equal 32 less-equal 32

route-policy oti deny node 10
if-match tag 200

route-policy oti permit node 20
if-match ip-prefix 32
apply tag 100

isis 1
import-route ospf 1 inherit-cost route-policy oti

在P2上将ISIS的路由引入到OSPF协议，添加Tag为200，并拒绝Tag为100的ISIS路由，即从RR2上引入的ISIS路由。
ip ip-prefix 32 index 10 permit 172.16.0.0 16 greater-equal 32 less-equal 32

route-policy ito deny node 10
if-match tag 100

route-policy ito permit node 20
if-match ip-prefix 32
apply tag 200

ospf 1
import isis 1 type 1 route-policy ito

在RR2上将Tag为200的路由优先级配置为150，即从P2上引入到OSPF协议中的ISIS路由。
route-policy pre permit node 10
if-match tag 200
apple preference 150

ospf 1
preference ase route-policy pre 10

在P2上将OSPF的路由引入到ISIS协议，添加Tag为300，并拒绝引入Tag为400的OSPF路由，即从RR2上引入到OSPF的路由。
route-policy oti deny node 10
if-match tag 400

route-policy oti permit node 20
if-match ip-prefix 32
apply tag 300

isis 1
import-route ospf 1 inherit-cost route-policy oti

在RR2上将ISIS的路由引入到OSPF协议，添加Tag为400，并拒绝Tag为300的ISIS路由，即从RR2上引入的ISI2路由。
route-policy ito deny node 10
if-match tag 300

route-policy ito permit node 20
if-match ip-prefix 32
apply tag 400

ospf 1
import isis 1 type 1 route-policy ito

在P2上将Tag为400的路由优先级配置为150，即从P2上引入到OSPF协议中的ISIS路由。
route-policy pre permit node 10
if-match tag 400
apple preference 150

ospf 1
preference ase route-policy pre 10

4.P1的ISIS进程：产生LSP的最大延迟时间是1S，初始延时为50ms，递增时间为50ms；使能LSP的快速扩散特性；SPF计算最大延迟为1S，初始延时为100ms，递增时间为100ms。

在P1上配置LSP的优化：
isis 1
timer lsp-generation 1 50 50
timer spf 1 100 100
flash-flood
