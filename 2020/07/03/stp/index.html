引入

冗余的网络产生的3个问题（二层环路导致的问题）

  1. 广播风暴，交换机转发BUM报文的机制导致（泛洪方式转发），数据帧没有防环机制导致环路无法消除。
    B：boradcast
    U：未知DMAC地址单播帧
    M：组播
  2. MAC地址表震荡，生成错误的MAC地址表项，导致转发发生错误。
  3. 重复数据帧接收，如果过存在大量的广播帧，导致网络性能下降甚至是不可用，终端设备也会消耗大量CPU资源处理广播。

STP的作用

  • 通过阻塞端口来消除环路，并能够实现链路备份的目的。

STP操作

  1. 选举一个根桥。
  2. 每个非根交换机选举一个根端口。
  3. 每个网段选举一个指定端口。
  4. 阻塞非根、非指定端口。
  • STP协议的所有角色都是通过比较BGPU的参数选举出来的。
  • STP定义每个交换机都有一个BID来标识自己。

BID = pri + mac

pri默认值32768
pri取值有两种标准

  1. 0-65535
  2. 0-61440 华为采用0-61440，但pri取值必须是4096的倍数。一共有16个值可取。

端口角色

根桥是发送最好BPDU的交换机

RP（根端口）：非根桥上接收最好BPDU的端口

DP（指定端口）：链路上发送最好BPDU的端口
DP端口要么对着RP，要么对着AP。

AP（备用端口）：既不是RP也不是DP的端口，被阻塞的端口 。

BPDU中4个选举参数

  • RID（Root Identifier）
    根桥的BID
  • RPC（Root Path Cost）
    对于根桥来说RPC永远等于0
    对于非根桥来说RPC是自己根端口到根桥的路径开销之和。
  • BID（Bridge Identifier）
    交换机自身的的BID
  • PID（Port Identifer）
    由端口优先级+端口id组成，优先级的取值为0-240，默认值128，取16的倍数。

什么是最好的BPDU？

  1. 比较RID，RID越小越优。
  2. 如果RID相同，则比较RPC路径和，越小越优
  3. 如果RPC路径和相同，则比较BID，越小越优
  4. 如果BID相同，则比较PID，越小越优
  5. 如果PID相同，则比较自身PID（特殊场景下：有hub的场景）

{4096_S1，0，4096_S1，128.4}

{根桥ID，累计根路径开销，发送者BID，发送端口PID}

STP的BPDU

  • 标准STP由根桥发送配置BPDU，非根桥是不能主动发送BPDU的。（华为优化：非根桥可主动发送BPDU，不用等根桥的BPDU到来）

  • 配置BPDU

    1. TCA配置BPDU
    2. TC配置BPDU

  • TCN BPDU

BDPU中 BPDU FLAG字段
TCA = 0 TC = 0 配置BPDU
TCA = 1 TC = 0 TCA配置BPDU
TCA = 0 TC = 1 TC配置BPDU
TCA = 1 TC = 1 TC/TCA配置BPDU

Message Age：已经存活的时间，根桥发出来是0，逐跳加1。当Message Age等于Max Age则认为该BPDU失效

Max Age：最大老化时间，默认20s

Hello Time：默认为2s，发送配置BPDU的时间间隔。

Forward Delay：默认为15s，转发时间间隔。

  1. 确定了监听和学习状态所需等待的时间。
  2. 收到TC BPDU后MAC地址表老化时间。

华为STP 根桥发送配置BPDU，非根桥也可以主动发送BPDU（华为从RSTP引入）

华为STP协议引入了大量RSTP的特点：

  1. 非根桥可以主动发送BDPU
  2. 收到次级BPDU立即老化
  3. 边缘端口

命令

只能在根桥上配，不然无效。

系统视图下：
stp timer hello
stp max-age
stp forward-delay

STP拓扑计算

STP算法实现的基本过程

  1. 初始状态

    由于每个桥都认为自己是根桥，所以在每个端口所发出的BPDU中，根桥字段都是用各自的BID，Root Path Cost字段是累计的到根桥的开销，发送者BID是自己的BID，端口PID是发送该BPDU端口的端口ID。

  2. 选择根桥

    网络初始化时，网络中所有的STP设备都认为自己是“根桥”，根桥ID为自身的设备ID。通过交换配置消息，设备之间比较根桥ID，网络中根桥ID最小的设备被选为根桥。

  3. 选择根端口和指定端口

  • 非根桥根据根端口的配置消息计算DP端口的配置消息。
  • 只要根端口确定了，它就会为其他端口计算配置消息，计算出的配置消息与其他端口进行比较，如果对比后发现计算出来的更加优，则替换掉，这些端口就成为DP端口并往外发BPDU。
  • 根据RP计算DP的BPDU参数，如果比指定端口的本身的BPDU参数好，则该端口为DP端口，如果没有比指定端口本身的BPDU参数好，则该端口为AP。

生成树拓扑的定位（工作中）

  1. STP首要工作定位根桥是哪台交换机。
display stp
STP协议的模式
交换机自身的BID
管理员配置的时间参数
交换机实际实施的时间参数
根桥的BID，如果该值和自身的BID相同，则该交换机为根桥/ERPC：这台交换机的根端口到根桥的开销（RPC）
对于STP协议来说无意义，用于MSTP中。
根端口
  1. RID最小的>RPC之和最小>BID最小的>BPDU中PID最小的>自己的PID最小的（场景：有hub的网络中）
  2. DP对端要么是RP要么AP，RP端口一定是DP
  3. 控制stp的拓扑：改RID、RPC、BID、PID

STP的端口开销

  • 标准：IEEE802.1d-1998、IEEE802.1t、华为计算方法。

  • 端口开销与端口速率、端口模式相关联。

命令

更改端口开销标准：stp pathcost-standard

更改端口开销：接口视图下stp cost

STP端口状态

端口状态 目的 说明
Forwarding 端口既转发用户流量也处理BPDU报文。 只有根端口或指定端口才能进入Forwarding状态。
Learning 设备会根据收到的用户流量构建MAC地址表，但不转发用户流量。 过渡状态，增加Learning状态防止临时环路。
Listening 确定端口角色，将选举出根桥、根端口和指定端口。 过渡状态。
Blocking 端口仅仅接收并处理BPDU，不转发用户流量。 阻塞端口的最终状态。
Disabled 端口不仅不处理BPDU报文，也不转发用户流量。 端口状态为Down。

Forwarding：转发数据，学习MAC地址，处理BPDU

Learning：不转发数据，学习MAC地址，处理BPDU

Listening：不转发数据，不学习MAC地址，处理BPDU

Blocking（discarding）：不转发数据，不学习MAC地址，处理BPDU

Disable：接口被禁用

稳定STP网络只有两种状态：即forwarding和blocking。

端口状态转换

图

一个选择为RP或者DP端口的并处于blocking状态至少需要30s进入到forwarding。

  1. 非根桥的RP失效，并存在AP时，AP成为新的RP，需要经过30s（Listening15s，Learning15s）才进入转发状态。
  2. 非根桥的RP失效，但不存在AP时，此时交换机认为自己失去了到根桥的所有路径。于是该交换机将自身作为根桥，发送以自己为根BPDU。下游交换机需要等待20s（Max age）将之前的BPDU老化，再处理该BPDU，此时需要50s恢复网络。（华为有优化，不等20s。直接处理。）
  3. ROOT失效，重新选举根桥，网络需要50s的恢复连接。

拓扑改变导致MAC地址表错误

什么时候生成树协议会认为拓扑发生了变化？
STP协议最早认为任何端口进入到forwarding状态或者进入disable状态时，认为STP拓扑发生变化。
后来STP对拓扑发生变化的定义做出了优化

  1. 当端口进入到forwarding状态认为拓扑发生变化，边缘端口除外。
  2. RP失效或者是RP进入forwarding状态会认为拓扑发生变化。
  3. DP端口失效不认为拓扑发生变化。

华为：
STP协议当接口UP并过度到forwarding状态，会发送TCN BPDU，
逐跳传给根桥，上游向下游发送TCA=1，TC=1的配置BPDU，用于确认TCN，并将TCN逐跳传给根桥。
根桥在接收到TCN的端口上回送一个TCA=1，TC=1的配置BPDU，用于确认TCN，并通知为网络拓扑发生变化。
配置BPDU，在其他端口上则发送TCA=0，TC=1的配置BPDU。

RP恢复到forwarding状态后，并且由一个指定端口在转发状态，此RP也会发TCN BPDU。

  • MAC地址表项的默认老化时间是300秒

TCN BPDU

BPDU通过组播工作

  • 拓扑改变通知（TCN）

  • 上游收到TCN BPDU后回送TCA BPDU（拓扑改变确认）并继续向上游发送TCN BPDU直到到达根桥，根桥收到后向下游回送TC置位的BPDU，传遍全网。

  • 全网的交换机只要收到TC置位的BPDU，就会把自己边缘端口之外的其他端口的MAC地址老化时间置为15s。

  • 发送TCN一般是AP与RP，是去往根桥的端口。发送TCA的一般是DP。首次发送TC置位的BPDU（配置BPDU报文中Flags字段的TC位置1）一般是根桥的DP，接着便是其他交换机的DP发送。

STP协议MAC地址表刷新机制：
当交换机收到TC BPDU后将自身所有端口（边缘端口除外）的MAC地址表的老化时间强制设置为15s。

TC BPDU会持续由根桥发送35s的时间，这段时间网络不稳定。所以需要这35s的时间，防止学习到错误的MAC地址。

边缘端口标准STP是不支持的，但是华为的STP协议支持边缘端口的设置。

STP 边缘端口

  • 由于TCN BPDU会导致大量的MAC地址提前老化（包括正确的MAC地址），导致大量未知单播帧的泛洪。可以利用边缘端口的机制来减少未知单播帧的泛洪。

  • 可在连接路由器、防火墙、终端、打印机、扫描仪之类设备的接口配置。

命令

接口视图下：stp edged-port enable
