IPv6地址结构

IPv6地址分为两部分

网络前缀，相当于IPv4中的网络ID
接口标识，相当于IPv4中的主机ID

接口标识生成方法

手工配置
系统通过软件自动生成
IEEE EUI-64规范自动生成

EUI-64前缀不足64的话就在前面补0，不能超过64。

EUI-64的生成：
根据接口的MAC地址生成EUI-64规范的interface id

2003:1:1:1: 02e0:fcff:fed5:1df9

48bit的MAC地址：00e0:fcd5:1df9 →中间插入fffe→ 00e0:fcff:fed5:1df9

然后将生成的EUI的第七个bit至反 0000 0000 ….→第7个bit置反→ 0000 0010

将接口MAC地址从中间插入固定长度16bit的FFFE，转成IPv6的格式，将MAC地址的第七bit从0转换成1。

EUI-64适用于终端设备。

第7bit的转换有两种转换方案：
1、如果是0则变成1，如果是1则不变（思科）
2、如果是0则变成1，如果是1则变成0（华为）

单播MAC地址中，第一个byte的第七bit是U/L，也称G/L，用于表示MAC地址的唯一性。为0是全局管理地址，为1是本地管理地址。在EUI中含义正好相反。所以要在意义上保持一致性。

IPv6地址的分类

IPv6地址分类

单播地址

  • 标识一个接口，目的为单播地址的报文会被送到被标识的接口

组播地址

  • 标识多个接口，目的为组播地址的报文会被送到被标识的所有接口

任意播地址

  • 标识多个接口，目的为任播地址的报文会被送到最近的一个被标识接口，最近节点是由路由协议来定义的
  • 任播地址与单播地址使用同一个地址空间

链路本地单播地址

FE80::/10

链路本地地址：FE80::/10 只用于链路上IPv6接口间的通信，不具备路由能力，凡是发向链路本地地址报文都不能被路由器路由。

凡是启用了IPv6协议栈的接口都会自动生成一个链路本地地址，默认前缀长度为64，默认前缀为FE80：：
interface id由EUI-64的规范自动生成，操作系统可能采用私有算法生成interface id。

链路本地地址的获取：
1、动态链路本地地址 EUI-64 （自己配前缀）
2、随机生成接口ID
3、静态链路本地地址
由于路由协议（RIPng，ospfv3）都使用链路本地地址建立邻居关系和发送协议报文，所以建议手动配置。容易识别。

链路本地地址的用途
1、ripng、ospfv3使用链路本地地址工作
2、icmpv6协议某些机制也使用

好处：ipv6的链路具备了天生的通信能力，无需向ipv4的接口一样必须配置ipv4地址才具备3层链路通信的能力

所以链路本地地址广泛用于邻居发现，无状态地址配置等应用

以链路本地地址为源地址或目的地址的IPv6报文不会被路由设备转发到其他链路

全球单播地址

2000::/3

全球单播地址

  • 类似IPv4公网地址
  • 由前缀、子网ID和接口标识组成
  • 一个接口可以配置多个全局单播地址，可以位于同一个子网或不同子网中

ISP分配给客户的前缀通常建议是 /48
企业的subnet id建议长度是 /64
interface id /64

全局单播地址获取方式
1、手动配置IPv6地址
静态配置前缀和接口ID
静态配置前缀和通过EUI-64获取接口ID
手动配置无编号IPv6地址（借地址）

2、动态配置IPv6地址
无状态自动配置（SLAAC）：利用该配置方法时，前缀和前缀长度是通过ICMPv6中的ND（邻居发现）路由器通告消息来确定，而接口ID则通过EUI-64进程创建
DHCPv6：类似于IPv4中的DHCP，通过DHCPv6服务器来实现

区别
SLAAC 无状态地址配置 分配地址的设备，无需保存和维护客户端信息
DHCPv6 有状态地址配置 分配地址的设备需要保存和维护客户端的个体信息

ICMPv6

概述

  • ICMPv6是IPv6的基础协议之一，用于源节点传递报文转发的信息或者错误。

  • 协议类型号（即IPv6 Next Header）为58。

  • 在IPv6中，ICMPv6除了提供ICMPv4的对应功能之外，还有其他一些功能的基础，如邻居发现、无状态地址配置、重复地址检测、PMTU发现等。

NS报文源地址的特点：
1、地址解析 使用自身接口的单播IPv6地址，目标地址请求节点组播地址。
2、NUD（邻居可达检测）使用自身link-local地址，目标单播地址。
3、DAD 使用：：，目的地址请求节点组播地址。

路由器发现概述

RA

发送机制：

链路上的路由器会定期发送RA消息

收到RA的主机将加入默认路由器列表中
收到RA的路由器将检查RA内容的一致性

RS

发送机制：

1、开机后主动发
2、路由器发送RA报文后主机回RS报文

重定向过程

PMTU

分片这个动作在IPv6中只有发的那个主机能做
重组这个动作只有在收的那个主机能做

分片重组会加大沿途路由器的负担

为了解决这个问题就需要发现路径上最小的MTU

IPv6过度技术

手动隧道：
1、隧道目的需要手动配置，都是点到点的隧道。
2、支持运行IPv6路由协议
3、支持星型和全互联组网
4、手动隧道配置较繁琐，需要创建大量隧道接口

ISATAP

ISATAP地址的interface id怎么生成？
主机使用固定32bit 标识0200:5EFE和自身网卡的IPv4组成。

路由器使用固定32it表示0000:5EFE和接口的IPv4组成。
