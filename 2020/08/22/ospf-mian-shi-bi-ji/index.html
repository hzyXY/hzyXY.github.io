OSPF面试笔记

1、影响OSPF邻居建立的过程中影响建立的因素有哪些：

  1. Router id不能相同
  2. 区域ID不同 /SPF算法基于区域，直连接口区域不同怎么算？/
  3. 在MA网络接口的地址掩码不同（P2P网络接口不检查掩码） /DR产生的2类LSA如果接口掩码与2类LSA的掩码不同，那么这些路由器的路由则无法不通/
  4. 认证类型不同
  5. option字段E位和N位不同
  6. hello interval配置不同
  7. dead interval配置不同
  8. 接口IP MTU不同（华为默认将DD报文的MTU设置为0，并且不做DD报文的MTU值比较，如果过使用ospf mtu-enable，则DD的MTU被设置为接口的IP MTU大小，并且在本接口将对接受的DD报文做MTU的大小的检测）
  9. 帧中继map映射缺少broadcast关键字

2、ospf邻居无法建立有哪几种可能性，如何解决？如果有相同的router-id会产生什么样的后果，为什么？

  1. 如果直连的两台路由器有相同的router-id会产生什么样的后果？为什么？
    产生的后果：如果直连路由器的Router-id冲突（ENSP实验现象：完全看不到对面的状态，init都没有）。

    为什么：

    1. OSPF邻居关系建立时会发送hello报文，hello报文中携带者router-id的信息，路由器接收到hello报文后发现与自身的hello报文相同就不在进行下一步了。
    2. 如果到达了Ex-start状态，那么将会进行主从选举，是通过比较Router-id的大小来决定LSDB同步时DD报文的序列号的，如果Router-id相同则无法进行主从选举，邻接关系则无法建立，如果可以，那么在LSDB中又无法区分这两台路由器所产生的1类LSA。

  1. 如果不直连的2台路由器有相同的Router-id会产生什么样的后果（同区域），为什么？
    产生的后果：路由器无法区分这两条不同的LSA，从而无法正确的计算路由。

    为什么：

    1. OSPF协议的1类LSA的link id通过router-id来表示，同时adv router也是由router-id来描述，当LSA的类型都是类型1时，导致路由器无法区分这两条不同的LSA，因为在LSDB中式通过link-id，adv router、lsa type来唯一区分一条LSA的，所以要保证同一个区域内每台路由器router-id不相同
    2. 与DR建立了full的邻居状态后，那么他们发送给DR的1类LSA中的三要素（Lsa type、LinkState ID AdvRouter）将会相同，这将无法区分这些LSA是谁产生的。（ENSP实验：在DR中不断查看此LSA这LSA会不断地在这两个Router-id冲突的路由器中刷新）

  2. 如果两台路由器有相同的router-id会产生什么样的后果？（一个在area 0、一个在area 1）为什么？
    产生的后果：路由器还是可以正常计算。

    为什么：
    理论上OSPF协议要求在同一个区域中的所有路由器router-id要唯一，但如果在不同区域中存在router-id的冲突，这并不会影响到区域内的 LSDB同步，因为类型1和类型2的LSA只能在本区域内泛洪，在LSDB中，LSA1和LSA2是基于区域来存放的，所以即使在不同区域中出现Link-id、adv router、lsa type一样的LSA，ABR依然可以正确的进行LSDB同步和进行基于区域的SPF算法计算内部的路由，而区域间的路由由ABR产生类型3的LSA实现，类型3的LSA并不包含起源路由器的router-id信息，而是ABR的router-id。其他区域路由器只需要计算自身到ABR的最短SPF树，即可计算去往区域间的路由，所以并不会影响区域间路由的计算。

  3. 如果2台路由器有相同router-id会产生什么样的后果？（一个在area2，一个人在are1，area0连接area1和area2）为什么？

    1. 类型3的LSA：每经过一台ABR的时候，adv router都会做相应改变，故在这种情况下，也不会影响域间路由的计算。及只许保证在同一个区域内的路由器route-id唯一即可。

    2. 类型5的LSA：如果OSPF网络中存在ASBR，那么ASBR的router-id一定要唯一，因为LSA5的adv router是由ASBR的router-id来代表的，并且LSA5在整个OSPF网络中泛洪，并且在泛洪过程中，adv router并不会改变，而且LSA5在LSDB中跟区域不存在关联。

      某台路由器自身的router-id和ASBR的router-id冲突，将会把这些LSA5的age时间设置成3600，并且进行泛洪（seq不变，只是将age时间设置为3600秒），希望其他路由器也将能删除这些LSA5包括产生这条LSA5的ASBR（毕竟这不是它自己产生的），这将导致产生这些LSA5的ASBR不断删除这些LSA，并又不断产生LSA5并进行泛洪，导致网络上反复产生LSA5又不断删除LSA5的行为，影响所有路由器的LSDB的稳定性，并可能产生网络震荡。

LSA

  1. 删除一条LSA，就是将LSA的age设置为3600s和metric设置为最大泛洪出去，通告这条LSA失效，并从LSDB中删除。
  2. 新旧比较次序：当路由器收到多个相同的LSA的时候，需要进行新旧的比较，放到自己的LSDB中：
    1. 先比较序列号，如果序列号越高，说明LSA越新
    2. 如果序列号相同则比较校验和，校验和的值越大，说明越新
    3. 如果校验和相同，则比较存活时间，如果只有一条LSA拥有大小为最大生存时间的老化时间，则认为该LSA是最新的。（要最老的或要删掉的那个！！）
    4. 如果这些LSA的老化时间之间的差别多于15min（称作MaxAgeDiff），那么拥有较小老化时间的LSA将是最新的LSA。（要年轻的）
    5. 如果上述条件都一样的话，那么则认为这两条LSA是相同的。

特殊区域的置位

ospf的特殊区域有stub和nssa，分别对应如下情况：
stub区域：E bit = 0 | N bit = 0
nssa区域：E bit = 0 | N bit =1
normal区域：E bit = 1 | N bit = 0
E代表是否支持5类LSA是否进行同步，N代表7类LSA是否要进行同步。
