BGP

  • BGP协议不产生路由，只用来传输路由
  • BGP只会传输自己”正在使用“的路由
  • BGP是一个路径矢量路由协议，并且是一个无环路径；AS-PATH在一个AS内部以及AS之间都有不同的防环机制。
  • BGP是一个应用程协议，使用的是TCP协议，端口号为179，单播。是唯一使用TCP的路由协议
  • 可以不停的可靠的发送大量的路由报文，利用的就是TCP的可靠以及滑动窗口。说明为什么BGP适合于庞大的网络，或者是internet网络，其他的协议即使可靠的但是他们都是通过一堆的ACK来做到可靠机制的。

为什么BGP要用TCP来工作？
1.效率高：TCP有窗口调整机制，能够更高效地传递路由信息。AS之间有大量地路由信息要传递。
2.可靠性高：有重传机制
3.TCP会话可实现长距离跨链路的信息传递。只要对方IP地址可达。传递路由更加灵活，不受逐跳传递路由的限制。

BGP状态机

图

Idle（空闲）

idle是BGP连接的第一个状态，在空闲状态，BGP在等待一个启动时间，启动事件出现以后，BGP初始化资源，复位连接重试计时器（Connect-Retry），发起一条TCP连接，同时转入Connect（连接）状态。路由器在路由表中搜索去往neighbor指定的IP地址的路由，看是否有前往该地址的路由，如果过没有明细的话，就会卡在这个状态，即使存在默认路由也是不可以的，因此此时不会主动的发起TCP连接，如果出现这样的情况，可以去查看show ip route，然后通过配置”静态路由“或者”通过IGP来通告“neighbor地址的IP来解决这个问题

长期停留在idle状态可能的原因：

  1. 没有去往邻居地址的路由，无法发起TCP三次握手，也无法完成TCP的3次握手。
  2. 邻居发起的TCP握手，被本端拒绝，拒绝的原因是邻居发起的TCP报文的源地址，不是本端指定的邻居地址。

peer x.x.x.x as-nu y 含义

1.向此邻居发送BGP报文的目标地址为x.x.x.x
2.从邻居所属的AS为y
3.此邻居向“我”发送BGP报文的源地址为x.x.x.x
4.自身去往x.x.x.x的路由的出接口地址，做为我向x.x.x.x发送BGP报文的源地址。

peer x.x.x.x connect-interface loopBack0
向x.x.x.x发送BGP报文的源地址设置为自身loopBack0接口地址

peer x.x.x.x ebgp-max-hop 10
建立EBGP邻居关系时，将BGP报文的ttl值，设置为10，当EBGP邻居跨链路进行建立时使用

IBGP邻居关系建立时，BGP报文的TTL=255，无需修改

Connect（连接）

在Connect状态，BGP发起的第一个TCP连接，如果过连接重试计时器（Connect-Retry）超时，就重新发起TCP连接，并继续保持在Connect状态，如果TCP连接成功，就转入OpenSent状态，如果过TCP连接失败，就转入Active状态。
如果过搜索到了，则开始建立TCP连接，如果成功的话，邻居状态则进入“Connect”状态。要分清楚是建立TCP连接的源地址和目标地址是什么，是双方的neighbor如果双方指定的IP地址不相同，那肯定就得不到“期望”的TCP连接。

当发起TCP连接的路由器，无法收到邻居TCP回应报文时，就会卡在Connect状态，并且5秒后重传一次TCP连接请求，再等待30秒左右，重新再次发起TCP连接请求，通常是由于发起端存在邻居地址的路由，并能主动发送TCP3次握手，但无法从邻居接收第二次握手的报文。通常是由于邻居缺乏到对端的路由，或者是应答报文中途被丢弃。

发起TCP连接的路由器，在5s内，如果没有得到邻居的任何回应报文，重传tcp报文，继续等待25s，还没有收到邻居的任何回应，会在重传TCP报文，在建立不起来重连。

Active（活跃）

在Active状态，BGP总是在试图建立TCP连接，如果过连接重试计时器（Connect-Retry）超时，就退回到Connect状态，如果过TCP连接成功，就转入OpenSent状态，如果过TCP连接失败，就继续保持在Active状态，并继续发起TCP连接，没有收到对open-send报文的确认报文。

当发送TCP连接路由器，能收到相应的邻居应答报文，代表2台路由能正常交互报文，但是依然无法建立起TCP的3次握手，将进入到Active。

OpenSent（打开消息已发送）

在OpenSent状态，TCP连接已经建立，BGP也已经发送了第一个Open报文，剩下的工作，BGP就在等在其对等体发送Open报文。并对收到的Open报文进行正确性检查，如果过有错误，系统就会发送一条出错通知消息并退回到idle状态，如果过没有错误，BGP就开始发送Keepalive报文，并复位Keepalive计时器，开始计时。同时转入OpenConfirm状态。

TCP连接建立之后，开始发送open报文，其中包含很多参数，以对方取根据这个“OPEN”来协商参数。

只要TCP建立成功，一定就可以发送出open报文，除非在接口做策略控制。

OpenConfirm（打开消息确认）

在OpenConfirm状态，BGP发送一个Keepalive报文，同时复位保持计时器，如果过收到了一个Keepalive报文，就转入Established阶段，BGP邻居关系就建立起来了。如果TCP连接中断，就退回idle状态。

如果过参数协商成功，就会发送keepalive报文，进行隐式确认，收到该报文后，就进入到open-confirm状态。

如果过在5s之内依然是没有收到Keep-alive报文的话，就进入到active状态。

不同意open-send报文中的内容，即参数协商不成功，就不会发送给open-confirm回去；一般就是AS号不同，此时邻居表中看到的状态是：不停的idle和active状态之间翻滚。

BGP对等体交互路由原则

只将最优路由发布给对等体 最优：>

  1. 从IBGP得到的路由，只能发布给EBGP对等体
  2. 从EBGP得到的路由，发布给EBGP和IBGP对等体
  3. 自身local起源的BGP路由，只会传给所有的IBGP邻居和EBGP邻居
  4. 只发送更新的BGP路由

BGP路由下一跳的特点

  1. 自身起源的BGP路由的下一跳为向邻居发送BGP报文的源地址。
  2. 通过EBGP邻居得到的路由在传递给IBGP邻居时，下一跳默认不变。
  3. 通过IBGP邻居得到的路由在传递给EBGP邻居时，下一跳改为自身向EBGP邻居发送BGP报文的源地址。
  4. 从EBGP邻居得到的路由，传递给成员EBGP邻居时，默认下一跳不变，可以使用next-hop-local进行修改。
    从IBGP邻居得到的BGP路由，传递给联盟内的EBGP邻居时，默认下一跳不变，也无法使用next-hop-local修改下一跳属性。
    从IBGP邻居得到的路由，传递给成员EBGP时，默认下一跳不变，无法使用next-hop-local进行修改。
    （华为设备默认规则，思科可以在这两种情况下都可以修改）
  5. 从成员EBGP邻居得到的路由，传递给成员IBGP邻居时，默认下一跳不变，无法使用next-hop-local进行修改。
  6. 第三方下一跳优化
    在同一个广播域中存在IBG邻居和EBGP邻居，通过IBGP邻居得到的路由传递给EBGP邻居下一跳默认不变。（并不常见）
    如果过加了peer x.x.x.x next-hop-local则会变。（但不适合）
  7. peer x.x.x.x next-hop-local将EBGP邻居得到的路由，发送给IBGP邻居x.x.x.x时，下一跳修改为向此IBGP邻居发送BGP报文的源地址。此命令在IBGP邻居之间生效。
  8. 通过RR反射的BGP路由，下一跳不会改变，使用next-hop-local也不会。

什么叫BGP反射的路由？

打破IBGP水平分割规则被RR传递的路由，称之为反射的路由。

解决BGP黑洞

  1. 在缺少BGP路由的路由器上，添加这些BGP路由的静态路由
    缺点：缺乏扩展性，配置工作量极大，不方便维护，不适用于大型网络。
  2. 将BGP路由引入到AS内部的IGP协议中
    优点：实施简单，配置工作量很少
    缺点：1.有环路风险。2.IGP协议管理路由的能力有限，IGP路由算法复杂，计算大量路由时，比较消耗设备资源。3.BGP路由出现震荡，对IGP协议的稳定性也会产生附带影响。4.IGP协议控制路由的能力有限，通常只能根据cost进行选路控制，在AS内部无法灵活进行路径的控制。
  3. AS内部的路由器都运行BGP协议，建立IBGP邻居
    优点：1.IGP路由和BGP路由分层工作，互不影响，网络的稳定性增强。2.每台路由器都可以根据自身的需要进行灵活的路由选路控制，方便进行路由的管理和控制。
    缺点：每台路由器都要进行BGP协议的配置，IBGP邻居数量多时，配置工作量比较大。每台路由器需要维护的TCP会话数量比较多，较消耗设备资源。
  4. AS边缘路由器在IBGP邻居之间建立LSP建立隧道
  5. AS边缘路由器在IBGP邻居之间建立GRE隧道

优点：利用隧道来解决BGP路由黑洞，AS内部路由器无需维护大量的BGP路由信息，对设备资源要求不高。配置相对简单。
缺点：1.隧道建好后，报文只能在隧道路径中进行转发，路径管理和控制能力较弱。2.如果是GRE隧道，由于它是一种点到点的静态隧道，所以扩展性比较差，通常可以由LSP隧道来代替。3.LSP隧道是动态的隧道，建立和维护都可以自动完成，只需要全网运行LDP协议进行隧道的建立 。

BGP反射器

不同角色的对等体

IBGP对等体可以有三种角色

  1. 路由反射器（Route Reflector）
  2. 客户机（Client）
  3. 非客户机（Non-Client）
对等体之间的关系

Client只需维护与RR之间的IBGP会话

RR与RR之间需要建立IBGP的全互连

Non-Client与Non-Client之间需要建立IBGP全互连

RR与Non-Client之间需要建立IBGP全互联

路由反射宣告原则
  1. 从非客户机得到的路由只会传给客户机，不会传给非客户机
  2. 从客户机得到的路由，传给客户机与非客户机

BGP路由属性

BGP路由属性是一套参数，它是对特定的路由进一步的描述

  • 公认必遵
    所有BGP路由器可以识别，且必须存在Update消息中
    如果缺少这种属性，路由信息就会出错
  • 公认任意
    所有BGP路由器都可以识别，但不要求必须存在Update消息中
    即就算缺少这类属性，路由信息也不会出错
  • 可选过度
    在AS之间具有可传递性的属性
    BGP路由器可以不支持此属性，但它仍然会接收这类属性，并传递给其他对等体
  • 可选非过度
    如果过BGP路由器不支持此属性，则相应的这类属性会被忽略，且不会传递给其他对等体

常用于修改BGP路由属性的方法

  1. network 引用route-policy 修改本地起源BGP路由的BGP属性
  2. import 引用route-policy 修改本地起源BGP路由的BGP属性
  3. peer export&import 引用route-policy 修改发送给BGP邻居或者从BGP邻居接收的BGP路由属性。
  4. aggregate 引用attribute-policy修改手动汇总的BGP路由属性

export import 方向上引用route-policy 时，要注意不匹配node的路由，默认被拒绝发送或则接收。

BGP选择路由的策略

当到达同一目的存在多条路时，BGP依次对比下列属性来选择路由

一、preferred-value（协议首选值)

默认情况下华为设备通过network，import，aggregate命令引入BGP表中的路由，协议首选值为0，通过其他BGP邻居传来的BGP路由本地默认设置为0.只能是import方向设置协议首选值，适用于IBGP，EBGP，不能在export方向应用。

只在本地有效，不传出去

使用场景：控制出口链路

二、Local_Pref（本地优先级）

  • 公认任意属性

  • 如果协议首选值相同则比较Local_Pref

本地优先级，长度为32bit，值越大越好，该属性值可以在本地（本地AS）内传输，不能传输到AS外部，默认值为100

作用：一般在AS的多个边界出口路由器上进行配置，针对不同的路由设置不同的优先级，以实现让内部IBGP路由器选择一个更加合适的出口路由器。

去往同一目地地多条路由中设置本地优先级，在IBGP对等体之间进行传递，不会应用于EBGP对等体之间。不会传出自己地AS，但是在成员EBGP邻居中可以传递，不会传递到出联盟之外地AS，取值范围0-4294967295之间，值越大，表明路由越优先。

IBGP邻居之间在in或者out方向都可以对local_pre属性做修改
EBGP邻居之间只能在in方向做local_pre属性地修改
成员EBGP邻居之间可以在in或者out方向对local_pre属性做修改

缺省情况下从所有EBGP邻居收到地路径属性中包含local_prefernce缺省认为也是100，但BGP表中显示为空（适用华为设备）

缺省情况下本地起源的路由也不包含local_pefernce缺省认为也是100，BGP表显示为空。（适用华为设备）

如果显示为空则代表传输时没有携带此属性

华为default local_prefernce的作用

  1. 修改本地起源的bgp路由的默认值，但在bgp表中显示为空，在将此路由传递给ibgp邻居时local_pre的值根据此命令设置。
  2. 默认情况下local_pre属性不传递给ebgp邻居，此命令对从ebgp邻居接收到的路由设置一个local_pre值，本地bgp表中显示为空，并传递给其他的ibgp邻居。（EBGP邻居传递过来的路由有效）
  3. 本地起源路由，从ebgp传递来的路由，手动汇总的路由生效
  4. route-policy优先于default local_prefernce

修改本地优先级的方法：

针对特定的邻居适用route-policy

  1. 没有RR的情况下
    此时无论是针对EBGP邻居，还是针对IBGP邻居都可以进行更改。针对EBGP邻居时在入向进行修改
    针对IBGP邻居，成员EBGP邻居，针对出向与入向都可以进行修改

  1. 存在RR的情况（经过不完整测试者此规则针对local-pref，orgin，med属性都适用，其他属性没测试）
    针对EBGP邻居发来的路由，入向策略可以进行修改
    针对非客户端发送来的路由，RR上做入向策略可以对本地优先级修改
    针对非客户端发送来的路由，RR上做出向策略发向客户端时，不能修改本地优先级，从非客户端来的路由携带什么属性，到了客户端依然时那些属性，但发向联盟EBGP邻居时可以修改。
    针对非客户端发送来的路由，RR上做出向策略发向EBGP邻居（联盟EBGP邻居）时，可以修改
    针对非客户端发送来的路由，在客户端做入向策略，可以修改掉本地优先级。

    针对客户端发送来的路由，在RR上做入向策略，可以进行属性的修改，并且可以传递给客户端和非客户端。
    针对客户端发送来的路由，在RR上做出向的策略，发送非客户端时，不能修改该本地优先级
    针对客户端发送来的路由，在RR上做出向的策略，发送EBGP（联盟EBGP）能修改该本地优先级。
    针对客户端发送来的路由，在RR上出向策略，发向RR的客户端的时间，不能修改本地优先级。

注：RR在IBGP邻居路由器之间，只是用来简单的打破水平分割，当做一个转接点，将IBGP的路由没有任何变化的传递给自己的其他IBGP邻居；只是用来简单的转发，仅此而已。

总结：RR在入方向可以修改BGP路由属性，出方向对于反射的路由不能修改该BGP属性，非反射路由可以修改。

三、路由种类

依次优选手动聚合路由、自动聚合路由、network命令引入的路由、import-route命令引入的路由、从对等体学习的路由。

自动汇总仅对import-route的路由生效

四、AS_Path

优选AS路径（AS_Path）最短的路由

  • 公认必遵属性
  • 不仅用于选路还用于防环
  • IBGP之间不检查AS_Path、EBGP之间才检查

AS_Path：指的时一个BGP在传输过程中所经过的AS的个数，个数越小越好。

注意：只有当一条BGP路由离开一个AS 的时候，才会将自身的AS号添加在这条路由的AS_Path属性中。

作用：用于AS之间的防环！在接收一个EBGP路由（包括成员EBGP）时，检查其AS_Path属性，如果过在其中发现自己的AS号，说明之前曾经穿越过自己的AS，此时就不会接收该路由，达到防环的目的，IBGP邻居之间不做AS_Path属性的防环检测。

  1. BGP路由每经过一个AS，就会将这个AS号加入自己的AS_Path中
  2. 当一个路由器收到两条相同目的地的路由，它将选择AS路径最短的路由。
  3. 这个属性只能在EBGP邻居之间传路由时进行改动，对IBGP邻居传来的路由该不了。Cisco中）

华为设备在联盟EBGP，成员EBGP邻居，IBGP之间in和out都可以修改as_Path。

华为设备：

BGP路由器如果从联盟EBGP邻居接收到BGP路由时，会检测此路由的AS_Path中是否存在自身联盟AS号，如果存在拒绝此路由，不检测此AS_PATH是否存在自身的成员AS号，如果存在依然接收此路由。

成员EBGP邻居之间会对接收的路由AS_Path做检查，如果成员内AS_Path（小括号中的AS_Path）存在自身成员AS号，不接收此路由，如果小括号中存在自身联盟AS号，可以接收。

成员EBGP邻居之间如果AS_Path中（除开小括号中AS_Path）存在自身成员AS号，则不接收此路由，可以自身联盟AS号。（防盗用公AS号）

可以有联盟AS不能由成员AS

成员ebgp和成员ibgp之间只能对小括的AS_Path属性做修改.
apply as-path none overwirte 替换原有AS_Path
apply as-path add 在小括号中添加AS_Path
apply as-path overwirte 替换原有AS_Path，在小括号中覆盖AS-path

IBGP邻居之间（包括成员IBGP邻居）不做AS_Path属性的防环检测

对于EBGP邻居出方向上不可以清空，对于IBGP邻居出方向可清空

local-rib x.x.x.x as_path 100

route-policy none overwirte

adj-rib-out x.x.x.x as_path 200 100

additive：原有AS_Path基础之上添加AS_Path属性

overwrite：替代掉原有AS_Path属性

对收到的路由就开始做AS_Path的检测了

peer allow-as-loop用于接收EBGP（包括成员EBGP）邻居传递来的路由允许AS_Path中出现自身联盟AS号（或自身成员AS号）的次数，默认1次。只要整个AS_Path（小括号内和小括号外的总和）中出现的次数小于或者等于配置的允许出现的值，就会接收次路由。

联盟内小括号里面的AS号，在选路时，不计算到as-path长度里面。算0个

as-path-limit不接收AS_Path长度大于此命令定义的值的BGP路由，同时不能发送（adj-out-rib表）AS_Path长度大于此值的BGP路由，小括号中的AS_Path长度计算在内，在import方向如果用策略增加了AS_Path长度超过limit设定的值，还是可以接收。

通过AS_Path列表过滤路由

ip as-path-filter

正则表达中的元字符：

正则表达式由直接量和元字符组成，直接量就是文本字符，元字符式充当运算符的正则表达式字符。

^$
表示匹配的字符串为空，即AS_Path为空的路由，表示只匹配本AS内部起源路由

.*
表示匹配任意字符串，即AS_Path为任意，表示匹配所有路由

^100
表示匹配字符串开始为100，即AS_Path最左边AS前3位（最后一个AS）为100、1001、1002等，表示匹配AS100、1001、1002等邻居发送的路由。

^100_
表示匹配字符串开始为100后面为符号，即AS_Path最左边AS为100，表示匹配AS100邻居发送的路由，比较前一个表示，“_”的好处就体现出来了，它可以和用来帮助我们限制匹配单独的一个AS。

_100$ 100$
表示匹配字符串最后为100，即AS_Path最右边AS（起始AS）为100，表示匹配AS100始发的路由

_100_
表示字符串中间有100，即AS_Path中有100，表示匹配经过AS100的路由

中括号：匹配字符集合

中括号式你能指定单字符的范围。例如：
ip as-path access-list 22 permit ^85[012349]$
上述过滤器匹配包含单一AS号850、851、852、853、854，859的AS_Path

如果过字符的范围连续，你可以只指定序列中开始和结束的字符：
ip as-path access-list 23 permit ^85[0-5]$
上述过滤器匹配包含单一AS号850、851、852、853、854、855的AS_Path

否定：匹配除字符集和外其他任何字符
当在括号中使用加字符时,将否定括弧中指定的范围。结果是正则表达式将匹配范围外的任何内容。例如:
ip as-path access-list 24 permit ^85[^0-5]$
该过滤器除了多一个加字符以外与上一个过滤器类似,指定了“不是0-5” 。该正则表达式将匹配856-859范围内的单个AS号。

通配符:匹配任何单一字符
点(.)匹配任何单一字符。 单一字符可能是一个空的。 考虑下面的过滤器：
ip as-path access-list 24 permit ^85.
该过滤器匹配一个有AS号850-859开始的AS_ PATH。由于”.” 可以匹配一个空格 , AS号85也能匹配成功。

求或:匹配字符串集合中的一一个(851|158)
(|)用作表示一个”或”操作。即(|)任何一边的字都可以匹配。例如:
ip as-path access-list 25 permit ^(851|852)$
该过滤器匹配单个AS号, 851或852的AS_ PATH。你可以扩展“或”功能来做多于两个可能性的选择:
ip as-path access-list 26 permit ^(851|852|6341|53)$

选择字符:匹配一个可能存在也可能不存在的字符
问号(?)匹配字的0个或-一个实例。 例如
ip as-path access-list 27 permit ^(850)?$
该过滤器匹配一个单一-AS号850的AS_ PATH或者匹配一一个空的列表。注意这里的园括号,表示元字符特殊字符”?”应用在这个AS号上。如果表达式使用850? , 元字符特殊字符只应用在最后一个字符上。该表达式匹配85或者850。

重复：匹配许多字符的重复
你可以使用两个元字符特殊字符来匹配重复的词:星号(* )匹配0次或多次重复，加号( + )匹配1次或多次重复。例如:
ip as-path access-list 28 permit ^(85)*$
该过滤器AS_ PATH匹配空的AS列表或包含1个或多个的AS号850的列表。即AS路径可以是(85)，( 8585)等。
ip as-path access-list 29 permit ^(850)+$

分界线：描述多个字
分界线(_ )用定的一串字。例如假设你想匹配指定的AS_ PATH ( 5610, 148 , 284, 13)。过滤器可以如下所示：
ip as-path access-list 30 permit ^5610_ 148 284 13$
下划线匹配行的开始、行的结束、逗号或空格。注意下面过滤器与上述过滤器的区别：
ip as-path access-list 30 permit. 5610 148_ 284. 13_
由于第一个过滤器指定了行的开始与结束,只有AS_ PATH (5610,148,284,13)才能匹配。第二个过滤器中，只要求指定的序列包含在AS_ PATH中,并不要求是唯一-的属性。 所以
AS_ PATH(5610, 148, 284, 13)，(23, 15, 5610, 148, 284, 13)和(5610, 148 , 284, 13, 3005 )都能匹配。
