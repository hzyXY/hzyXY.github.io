OSPF

  • 开放式最短路径优先OSPF（Open Shortest Path First）是IETF组织开发的一个基于链路状态的内部网关协议（Interior Gateway Protocol）
  • OSPF基于IP协议，协议号 89
  • 目前，OSPFv2仅支持IPv4，OSPFv3仅支持IPv6
  • COST值以出接口方向做累加

OSPF协议具有以下特点：

  • OSPF把自治系统AS（Autonomous System）划分成逻辑意义上的一个或多个区域；
  • OSPF通过LSA（Link State Advertisement）的形式发布路由；
  • OSPF依靠在OSPF区域内各设备间交互OSPF报文来达到路由信息的统一；
  • OSPF报文封装在IP报文内，可以采用单播或组播的形式发送。

OSPF拓扑和路由器类型

OSPF整体拓扑

  • OSPF把自治系统划分成逻辑意义上的一个或多个区域,所有其他区域必须与区域0相连。区域0称为骨干区域。

路由器类型

  • 区域内路由器（Internal Router）：该类设备的所有接口都属于同一个OSPF区域。

  • 骨干路由器（Backbone Router）：该类路由器至少有一个接口属于骨干区域。所有的ABR和位于Area0的内部路由器都是骨干路由器。

  • 区域边界路由器ABR（Area Border Router）：该类路由器可以同时属于两个以上的区域，但其中一个接口必须在骨干区域。ABR用来连接骨干区域和非骨干区域，它与骨干区域之间既可以是物理连接，也可以是逻辑上的连接。

  • 自治系统边界路由器ASBR（AS Boundary Router）：与其他AS交换路由信息的路由器称为ASBR。ASBR并没有特定的位置要求，它可能位于普通的区域，也可能是ABR。只要一台OSPF路由器引入了外部路由的信息，它就成为ASBR。

OSPF的报文类型

  • OSPF报文直接运行于IP之上，IP协议字段为89，OSPF有五种报文类型
  • OSPF报文的头部格式都是相同的
  • 除了Hello报文外，其他OSPF报文都携带LSA信息
报文类型 报文作用
Hello报文 周期性发送，用来发现和维持OSPF邻居关系
DD报文（Database Description packet） 描述本地LSDB（Link State Database）的摘要信息，用于两台设备进行数据库同步
LSR报文（Link State Request packet） 用于向对方请求所需的LSA。设备只有在OSPF邻居双方成功交换DD报文后才会向对方发出LSR报文
LSU报文（Link State Update packet） 用于向对方发送其所需要的LSA
LSAck报文（Link State Acknowledgment packet） 用来对收到的LSA进行确认

  3. auth type 一样
  4. auth data 一样
  5. hello interval 一样
  6. dead interval 一样
  7. option 中 N bit，E bit取值一致（区域类型决定取值）
建立邻居关系的进阶要求
P2P
  • 在P2P（PPP，HDLC）的网络类型邻居关系建立条件：P2P网络类型不对hello报文的掩码长度是否一致做检测
    • 地址在一个网段，但掩码不一致能建立邻接关系 比如：10.1.12.1/30 ——10.1.12.2/24
    • 不在一个网段能建立邻居关系，路由也能计算 1.1.1.1/24 ——2.2.2.2/30
    • 一个接口包含另一个接口的IP也能建立邻居关系，路由也能计算 10.1.12.1/30——10.1.12.10/24
广播型，NBMA网络类型
  • 广播型，NBMA网络类型要对hello报文的掩码长度做一致性检测
    • 如果不在一个网段，邻居关系无法建立
    • 如果不在一个网段，但一个接口包含另一个接口的IP，邻居关系无法建立
    • 如果链路地址都在一个网段，但彼此掩码不一致，邻居 关系无法建立
    • 如果链路地址都在一个网段，并且掩码相同，邻居关系可以正常建立
P2MP
  • P2MP（默认情况下IP地址必须在同一网段，并且掩码相同才能建立邻居关系，但可以通过命令[ospf p2mp-mask-ignore]忽略掩码的检测,但也要求在同一网段。）
    • 两边地址必须同一网段，因为DD，LSR，LSU报文在P2MP网络上是单播发送如果过不在一个网段，导致后续的DD，LSR，LSU无法发送
    • 在一个网络，但掩码不一致，邻居无法建立，使用ospf p2mp-mask-ignore可解决问题
    • 接口不在一个网段，无法建立邻居关系，使用ospf p2mp-mask-ignore无法解决问题
    • 接口不在一个网络，但接口包含另一个接口，无法建立邻居关系，使用ospf p2mp-mask-ignore无法解决问题
MTU对邻居关系的影响

如果过链路是PPP，在PPP上的链路会自动协商链路的MTU，使得链路两边的MTU值一致，取接口最小MTU值作为链路的MTU，默认DD报文的MTU为0，都小于链路的MTU，因此邻居关系正常建立，如果过使用了ospf mtu-enable，那么在DD报文中MTU值为链路协商后的MTU，因此还是保持一致，邻居也可以正常建立。

如果过链路是HDLC，由于缺乏MTU的协商机制，所以当两边的MTU不一致，并且用ospf mtu-enable时，如果过DD报文中的MTU值大于接收DD报文的接口本地的MTU值时，邻居关系无法正常建立，另据关系将卡在exstar的状态。

LSR报文

  • 从DBD报文中的LSA摘要信息中可得知自己缺少的LSA从而发送LSA请求报文请求所缺的LSA
  • 里面包含了所请求报文的三要素（有没有问题）

用于回应邻居的LSR报文中所缺少的LSA，LSU报文中包含了完整的LSA信息

  • 用于回应LSU中自己所接收的报文，仅有LSA摘要信息。

OSPF 邻居状态机

状态机 含义
Down 邻居会话的初始阶段。表明没有在邻居失效时间间隔内收到来自邻居设备的Hello报文。
Attempt 处于本状态时，定期向手工配置的邻居发送Hello报文。说明：Attempt状态只适用于NBMA类型的接口。
Init 本状态表示已经收到了邻居的Hello报文，但是对端并没有收到本端发送的Hello报文。
2-way 互为邻居。本状态表示双方互相收到了对端发送的Hello报文，建立了邻居关系。如果不形成邻接关系则邻居状态机就停留在此状态，否则进入Exstart状态。
Exstart 协商主/从关系。建立主/从关系主要是为了保证在后续的DD报文交换中能够有序的发送。初始DD序列号也是在此状态下决定的。在此状态下发送的DD报文不包含链路状态描述。
Exchange 交换DD报文。本端设备将本地的LSDB用DD报文来描述，并发给邻居设备。
Loading 正在同步LSDB。两端设备发送LSR报文向邻居请求对方的LSA，同步LSDB。
Full 建立邻接。两端设备的LSDB已同步，本端设备和邻居设备建立了邻接状态。

注意

  1. Drother之间会停留在2-way状态
  2. Exstar中协商主从关系的作用：，因为DD报文不周期性发送，且ospf协议是直接封装在IP报文之上的，IP协议是不保证可靠传递的。为了保证DD报文的可靠传输，ospf协议就设计主从选举的机制来保证后续DD报文的可靠传输。
  3. 所以router id不能冲突的原因：1.在Exstart状态中需要进行主从选举，选举的依据就是router id。2.产生的1类LSA三要素相同导致不能区分。
  4. 在Exchange状态中，DD报文的隐式确认：从路由器总是用主路由器的seq num向主路由器发送DD报文。主路由器总是用从路由器的DD的seq num+1向从路由器发送DD报文。隐式确认保证DD报文交互的可靠性。

OSPF LSA

LSA新旧对比

  1. 首先比较序列号（有32位，从0x80000001到0x7fffffff），如果序列号越高，则越新。
  2. 如果序列号相同，则比较校验和，校验和越高越新。
  3. 如果校验和相同，则比较存活时间（LS Age），如果这些LSA中只有一条存活时间为3600s，则认为该LSA是最新的。
  4. 如果所有LSA的存活时间都小于45分钟的话，则选择存活时间最小的放入自己的LSDB中。
  5. 如果上述条件都一样的话，则认为这两条LSA是相同的，将本地的LSA保留。

LSA头部

LSA类型

LSA类型 LSA作用
Router-LSA（Type1） 每个设备都会产生一条，描述了设备自身直连端口的链路状态和开销，只在所属的区域内传播。
Network-LSA（Type2） 由DR（Designated Router）产生，每个DR都会产生一条，用于描述广播型网络和NBMA网络， 这种LSA描述了所有连接着伪节点的路由器，只在所属的区域内传播。
Network-summary-LSA（Type3） 由ABR产生，描述区域内某一个网段的路由，并通告给其他区域（除了Totally Stub和Totally NSSA区域）。例如：ABR同时属于Area0和Area1，Area0内存在网段10.1.1.0，Area1内存在网段11.1.1.0，ABR为Area0生成到网段11.1.1.0的Type3 LSA；ABR为Area1生成到网段10.1.1.0的Type3 LSA。
ASBR-summary-LSA（Type4） 由ABR产生，描述本区域到其他区域中的ASBR的路由，通告给除ASBR所在区域的其他区域（除了Stub区域、Totally Stub、NSSA区域和Totally NSSA区域）。
AS-external-LSA（Type5） 由ASBR产生，描述到AS外部的路由，通告到所有的区域（除了Stub区域、Totally Stub、NSSA区域和Totally NSSA区域）。
NSSA LSA（Type7） 由ASBR产生，描述到AS外部的路由，仅在NSSA区域内传播。
Opaque LSA（Type9/Type10/Type11） Opaque LSA提供用于OSPF的扩展的通用机制。其中：Type9 LSA仅在接口所在网段范围内传播。用于支持GR的Grace LSA就是Type9 LSA的一种。Type10 LSA在区域内传播。用于支持TE的LSA就是Type10 LSA的一种。Type11 LSA在自治域内传播，目前还没有实际应用的例子。
Router-LSA（Type1）

LSA类型：Router LSA 路由器LSA
作用：用于说明自身哪些链路启用了OSPF以及这些链路的状态
产生路由器：每个运行了OSPF的路由器都会产生
LSA传输的范围：同一个区域内部
LinkState ID：自己本身的Router-id
AdvRouter：自己本身的Router-id
ADV是否变化：在传输过程中不会发生变化
Link-date：链路信息

Type：LSA类型为Router

LinkState ID：自己的Router ID

AdvRouter： 谁产生这条LSA的Router ID

类型1的LSA的Link type 分为4种

  1. stubnet：描述路由器自身直连网段的网络号以及掩码，开销。Link ID就是直连网络的网路号；data就是直连网络的子网掩码；metric就是自身直连网络的开销值。
  2. P-2-P：用于描述点到点链路上的邻居。Link ID就是邻居的Router id；data就是自身与邻居相连接口的地址；metric就是到邻居的开销。
  3. TransNet：用于描述和某个伪节点（当路由器看待）相连，代表此路由器存在一个广播型或是NBMA型的链路。link id是伪节点的标识符，用DR的接口的地址充当，data是与伪节点相连接口的地址，metric是到伪节点的开销。
    伪节点的作用：在广播，NBMA中虚拟出一个伪节点，简化网络复杂度，提升算法性能。
  4. Virtual：用于描述v-link上的邻居（p2p，link id是v-link对端路由器的router-id，data是本端接口地址，metric是v-link的开销。

结论：
凡是link type p2p link-id 都对应一个link state id相同的1类LSA
凡是link type trannet link id 都会由一个对应的link state id相同2类LSA

Network-LSA（Type2）

用于描述伪节点连接着那些路由器，由DR路由器产生

LSA类型：Net LSA 网络LSA
作用：用于说明DR的管理范围
产生的路由器：只能是DR
LSA的传输范围：一个区域内部
LinkState ID：DR路由器的接口地址
AdvRouter：DR的Router ID
AdvRouter是否变化：不会变化
Link-date:
DR路由器的接口上的子网掩码
伪节点连接节点的Router ID

Type：LSA类型为Network

LinkState ID：DR接口的IP地址

AdvRouter： DR的Router id

区域内路由通过1类LSA和2类LSA实行SPF算法得到，100%无环，区域内防环通过SPF算法实现。

Network-summary-LSA（Type3）

ABR路由器通过自身类型1的LSA的B比特位，表明自己是ABR
LSA类型：Net summary LSA网络汇总LSA sum-net
作用：通告域间路由
产生的路由器：ABR，区域边界路由器
LSA的传输范围：同一个区域内部
LinkState ID：区域间路由的前缀
AdvRouter：ABR的RID
AdvRouter是否变化：每经过一个ABR，都会发生变化
Link-date：域间路由的子网掩码

Type：LSA类型为Sum-net

LinkState ID：网络号

AdvRouter： ABR的Router id

Net mask：携带路由的掩码信息

Metric：ABR到此路由的开销

怎么判断一台路由是否是ABR？如果是ABR将会在拓扑数据库中的1类LSA中的Options字段中说明自己是ABR。在报文中是在一类LSA消息中的Flags字段中B bit位置位说明自身是ABR。

路由器 如何说明自身是ABR、ASBR或是否存在v-link
type1-lsa Flages
v bit 0代表路由器没有v-link 1代表此路由器上配置了v-link
E bit 0代表此路由器不是ASBR 1代表此路由器是ASBR
B bit 0代表此路由器不是ABR 1代表此路由器是ABR

ABR 将直连区域的的区域内路由（1，2LSA计算出来的路由），转化成其他区域的3类LSA（路由信息，描述ABR到这些路由的开销）

区域间路由计算：
ABR将区域0中的3类LSA转化成非骨干区域的3类LSA。
ABR将非骨干区域的3类LSA转化成其他非骨干区域的3类LSA
ABR不能将非骨干区域的3类LSA再次转化骨干区域的3类LSA，实现区域间的防环
区域0的3类LSA优于非骨干区域的3类LSA计算路由
ABR如果过能用1类计算出来的路由，就不会用3类来计算这条路由
自身到ABR的下一跳就是去往区域间路由的下一跳
自身到ABR的开销+ABR到区域间路由的开销等于自身到区域间路由的开销
ABR在区域0存在活动邻居时，不会考虑使用非骨干区域的3类LSA计算区域间的路由。原因是1类优于3类，骨干区域的3类优于非骨干区域的3类
在区域0中ABR产生某条路由的3类LSA后，不会使用其他ABR产生的这条路由的3类LSA计算路由

ASBR-summary-LSA（Type4）

如果存在ASBR，ASBR所在区域的ABR就会产生类型4的LSA，不一定需要有类型5的LSA出现
LSA类型：ASBR summary LSA AS边界路由器汇总LSA
作用：告诉其他区域的路由器如何到达ASBR
产生路由器：与ASBR在同一区域里面的ABR产生
LSA传输范围：一个区域内部
LinkState ID：ASBR的router id
AdvRouter：ABR的router id
AdvRouter是否变化：每经过一个ABR，都会发生变化一次
Link-date：子网掩码为0

4类LSA的产生：
由ASBR所在区域ABR向其他区域产生，如果过有多个ABR，都产生4类的LSA，用于描述自身到ASBR的开销

4类LSA防环以及选路机制

ABR通过1类LSA和4类LSA都能计算到ASBR的路径时，选择到ASBR开销小的路径，作为外部路由的路径，没有1类LSA优于4类LSA的规则。
ABR通过优先使用区域0的4类LSA计算外部路由，区域0的4类优于非骨干的4类
当ABR在区域0有活动邻居时，不会用非骨干区域的4类LSA，计算外部路由

AS-external-LSA（Type5）

ASBR路由器通过自身类型1的LSA中flag的E比特位，表明自己是ASBR，如果B=1,E=1表明路由器即是ABR，也是ASBR，如果过V=1表明此路由器存在v-link
LSA类型：ASBR external LSA
作用：用于通告外部路由
LSA的传输范围：整个OSPF域
LinkState ID：外部路由前缀
AdvRouter：ASBR的router-id
AdvRouter是否变化：在传输过程中不变
Link-date：子网掩码

外部路由计算方式，转化地址为0.0.0.0时
外部路由的cost=自身到达ASBR的cost+引入外部路由时的cost
下一跳：去往ASBR下一跳，为外部路由的下一跳。

外部路由计算方式，转化地址不为0.0.0.0时
外部路由的cos=自身到达FA地址的cost+引入外部路由时的cost
下一跳：去往FA下一跳，为外部路由的下一跳

FA地址作用：唯一作用实现OSPF网络路径优化

5类LSA什么时候FA地址不为0？不为0的时候，FA地址时什么？
如果同时满足以下3个条件，则FA地址不为0：

  1. 引入的这条外部路由，其对应的出接口启用了OSPF
  2. 引入的这条外部路由，其对应的出接口未设置为passive-interface（silent-interface）
  3. 引入的这条外部路由，其对应的出接口的OSPF网络类型为Broadcast或NBMA

FA地址是外部路由的下一跳为5类LSA的FA地址

metric type=2 在路由表中只显示引入时的外部路由开销，不体现AS内部所经过的路径开销计算在内
metric type=1 在路由表中显示引入时外部开销与在AS内部所经过的路径开销之和

4类和5类的能不能单独出现？可以
单区域的OSPF网络引入外部 路由时，不需要4类的LSA，存在5类LSA
空引入外部路由时或引入的外部路由在ASBR上被全部过滤时 ，会产生4类，而不存在5类

5类LSA如何防环？

5类自身没有防环机制
与ASBR同一区域的路由器可以通过SPF算法实现外部路由防环
没有与ASBR在同一个区域的路由器通过4类的LSA防环

区域间路由汇总

不能在区域内做路由汇总的原因：路由汇总针对的是路由条目，区域内的路由都是都是用1类2类LSA计算出来的，1类和2类LSA是无法做汇总，是要在区域内泛洪的。所以无法在区域内做路由汇总

ABR的汇总：只能针对直连区域的1类2类LSA产生的路由进行汇总，已经成为3类的LSA是无法进行ABR的汇总。

汇总路由的COST值，继承明细路由中COST值最大的

ABR产生汇总的3类LSA后，自身路由表不会汇总路由

明细路由都不存在时，汇总路由才不会产生

NSSA LSA（Type7）

7类LSA产生FA地址特点

1.如果和5类的LSA的FA地址生成条件相同，则FA地址也和5类FA地址相同
2.如果和5类的LSA的FA地址生成条件不同，则FA地址为ASBR上某个加入到OSPF进程的接口地址。（地址大的环回口优先，其次是物理接口）

为什么7类LSA引进来的都带有FA地址（七类默认的不带转发地址）？而5类LSA要在满足条件下才有转发地址？
有做7转5的advrouter是会变成abr的router id。如果不带转发地址，而去往abr的cost大于去往fa地址时就会产生次优路径。所以7类LSA都基本带有转发地址。

OSPF的网络类型

点到点P2P类型

当链路层协议是PPP、HDLC时，缺省情况下，OSPF人认为网络类型是P2P。在该类型的网络中，以组播形式（224.0.0.5）发送协议报文（Hello报文，DD报文，LSR报文，LSU报文、LSACK报文）。

P2P网络类型的特点

  1. hello报文时间为10s
  2. 组播更新
  3. 由于是组播更新，不用手动指定邻居，可以自动发现
  4. 3层是由广播的，2层也一定要有广播能力。（FR map后一定要加Broadcast）
  5. 不需要选举DR和BDR
  6. 不需要手工加MAP映射

点到多点P2MP类型（Point-to-Multipoint）

没有一种链路层协议会被缺省认为是Point-to-Multipoint类型。点到多点必须是由其他的网络类型强制更改的。常用做法是将非全连通的NBMA改为点到多点的网络。在该类型的网络中以组播形式（224.0.0.5）发送Hello报文，以单播形式发送其他协议报文（DD报文、LSR报文、LSU报文、LSACK报文）。邻居自动发现，无需选择DR，BDR。

生成帧中继的主机路由当成stubnet它的目的是为了什么？
为了能够在分支机构中不存在虚链路的情况下还能实现通信，而无需人工添加到达这些分支机构的静态映射，简化了配置。

NBMA类型（Non-broadcast multiple access）

当链路层协议是ATM，FR时，缺省情况下，OSPF认为网络类型是NBMA。在该类型的网络中，以单播形式发送协议报文（Hello报文、DD报文、LSR报文、LSU报文、LSACK报文）。邻居无法自动建立，手动建立邻居关系，需要选择DR，BDR。

NBMA是为了不支持组播广播能力的FR的全互联量身定做的

星型NBMA网络，FR map有brocast没brocast参数都可以。

  1. 中心做DR，不需要BDR，分支做DRother
  2. 邻居手动指定
  3. 伪节点算法，需要在分支之间彼此增加映射

结论：
星型FR网络，并不适用NBMA网络类型来组网。
全互联FR网络，适用NBMA网络类型来组网。

星型NBMA网络，修改为brocast的网络，前提是FR map有brocast参数即可。

  1. 中心做DR，不需要BDR，分支做DRother
  2. 邻居自动发现
  3. 伪节点算法，需要在分支之间彼此增加映射

星型的NBMA网络。使用P2MP是最合适组网。

广播类型

当链路层协议是Ethernet，FDDI时，缺省情况下，OSPF认为网络类型是Broadcast。在该类型的网络中，通常以组播形式发送Hello报文，LSU报文和LSACK报文。其中，224.0.0.5的组播地址为OSPF路由器的预留IP组播地址；224.0.0.6的组播地址为OSPF DR的预留IP组播地址。以单播形式发送DD报文和LSR报文。邻居可以自动建立，需要选择DR，BDR。

OSPF的防环

区域内路由防环

域内主要用SPF算法

一类LSA和二类LSA通过SPF算法进行防环

区域间路由防环

由于OSPF区域间路由计算本质上还是一种DV算法，无法通过SPF算法进行区域间的放环，所以其核心思想是，保证骨干区域连续，非骨干区域和骨干区域直接相连，非骨干区域之间的流量只能通过骨干区域传递，从而避免有过多的路由计算从而导致环路，具体规则有以下几种：

  • 不论cost值大小，LSA1和LSA2计算的路由优于LSA3计算的路由。
  • 优先使用骨干区域的LSA3计算路由，优于非骨干区域的LSA3。
  • 非骨干区域的LSA3不会再次传回骨干区域。
  • LSA3只能通过ABR进行泛洪，非ABR不会在区域间泛洪LSA3。

区域外路由防环

（1）LSA5在整个AS内部泛洪的时候link id，adv router，type都不会发生改变，特殊区域除外。
（2）与ASBR在同一区域的路由器可以直接通过SPF算法计算出一条无环的去往ASBR的路径。
（3）其它区域路由器是通过LSA4计算自身到达ASBR的路径和开销，而LSA4和LSA3的放环机制类似，其本质是描述ABR到达ASBR的开销，对于其他区域计算到达外部路由开销实际上是自身到达本区域ABR开销加上ABR到达ASBR开销加上ASBR到达外部路由开销，由于自身到达ABR的路径可以保证无环，而LSA4又和LSA3具有类似的放环机制，所以可以保证ABR到达ASBR无环，进而可以确定其他区域路由器到达外部路由是无环的。

OSPF的选路

LSA1 > LSA3（区域0） > LSA3(非骨干) > LSA5/LSA7(cost type 1 > cost typy 2)

LSA5和LSA7选路机制

转发度量值分为两种：
1.当转发地址为0.0.0.0，则转发度量值为到ASBR的cost
2.当转发地址不为0.0.0.0，则转发度量值为到转发地址的cost值

转发度量值：到ASBR的开销
种子度量值：ASBR引入外部路由的开销

当转发地址不同，但到转发地址的开销相同，则负载分担，5类7类都遵循此特点

Cost-type2

默认为类型2

转发地址为0.0.0.0或不为0.0.0.0都遵循以下规则：

如果引入种子开销相同，转发度量值相同，则负载分担
如果引入种子开销相同，转发度量值不同，则选择转发度量值小的优先
如果引入种子开销不同，转发度量值相同，则选择种子度量值小的优先
如果引入种子开销不同，转发度量值不同，则选择种子度量值小的优先

类型5或类型7的LSA，当转发地址非0时，优先使用种子开销小的LSA计算路由，如果过种子开销值相同，则优先选择到转发地址开销值小的LSA计算路由。如果过种子开销相同，转发地址也开销相同，转发地址相同或不同，则负载分担。
如果ASBR上去往外部同一条路由开销相同，则使用转发地址大的产生类型5或是类型7的LSA

总结
Cost-type2通过可以在不影响AS内部拓扑结构的情况下，选择“最佳的”出接口来使用。通常用于不关心内部转发路径时，选择一个“最佳的”路径，通常可以实现人为对业务出口的控制。

Cost-type1

转发地址为0.0.0.0或不为0.0.0.0都遵循以下规则：

如果过引入种子开销相同，转发度量值相同，则负载分担
其他情况根据种子开销和转发度量值之和，和值小的优先，如果过和值相同则负载分担。

Cost-type1总是比Cost-type2的优先，不区分是5类还是7类的LSA

类型5和类型7的LSA，当转发地址非0时，如果过种子开销相同，转发地址开销相同，转发地址相同，则负载分担。
类型5和类型7的LSA，当转发地址非0时，如果过种子开销相同，转发地址开销相同，转发地址不同，也负载分担。
类型5和类型7的LSA，使用最终计算出来的COST值最低的路由，作为最佳路由，不关心转发地址开销值和种子开销值。

OSPF的特殊区域

  • 特殊区域包括Stub区域，完全Stub区域，NSSA区域，完全NSSA区域

  • 共同点：不允许5类LSA进行泛洪

  • 作用：减少外部LSA的数量，节省内存和CPU资源，增强网络的稳定性。

Stub区域

不泛洪5类LSA，不产生4类LSA，会产生一条3类的LSA为默认路由

常规区域hello option E=1 N=1

stub区域Hello报文 option E=0 N=0

dd报文不携带5类LSA，4类LSA

内部路由器不可以变成ASBR

完全Stub区域

不泛洪5类LSA，不产生4类LSA，不通告明细3类LSA，只会产生一条默认的3类的LSA。

常规区域hello option E=1 N=1

完全stub区域Hello报文 option E=0 N=0

内部路由器不可以变成ASBR

NSSA区域

Not So Stubby Area 不那么完全的末节区域

允许本区域存在ASBR，不泛洪5类LSA，不产生4类LSA，会产生7类LSA，ABR还会产生一条七类的默认路由

当NSSA区域存在多个ABR，默认router id大的做7转5，泛洪到其他区域。不允许其他区域的5类LSA泛洪到NSSA区域,。

当存在多条7类LSA，在7转5的时候，只产生1条5类LSA，即将最优的7类进行7转5。

当到达转发地址的路由是负载分担，则计算的外部路由也会负载分担

当NSSA区域的ABR收到一条七类的LSA和一条5类的LSA时，并且转发地址相同时。优先使用7类的进行外部路由计算。可能因为能用1类LSA去到转发地址。

完全NSSA区域

允许本区域存在ASBR，不泛洪5类LSA，不产生4类LSA，不通告明细3类LSA，会产生7类LSA，完全NSSA区域的ABR会同时产生1条3类默认路由和一条七类的默认路由（3类优于7类）

OSPF虚链路

虚链路只能穿越一个区域

虚链路通过单播传递报文

虚链路不检测掩码和地址是否在同一个网段

即使交互报文时，源地址和目的地址完全不同，依然可以正常进行ospf报文交互

怎么检测虚链路的状态变化？

  1. 通过周期性发送hello包
  2. 通过transit区域的LSA1的变化，判断vlink是否可达，靠SPF算法，比hello机制更快。
  3. 直连接口失效

虚链路的使用场景

一般虚链路用于临时的连接或者备用的连接，设计中应该避免虚连接的出现，使用隧道技术可以在某些场景下代替虚链路，如GRE隧道，或者是通过不同的OSPF进程实现重引入。

  1. 用于连接非直连的非骨干区域
  2. 用于连接不连续的区域
  3. 用于两个非骨干区域的连接（思科不支持）
  4. 用于区域0的备用链路
  5. 路径优化时使用

OSPF虚链路的防环

虚链路的缺点：容易产生环路

  1. 通过虚链路的到的1类LSA所计算出的路由，不会转换成虚链路所在区域的3类LSA，目的是为了防环
  2. 无法再虚链路所在的区域对区域0的路由实现汇总
  3. 即存在vlink的ABR得到一条3类LSA（非自身产生的），那么该LSA在自身区域0中存在，此时也要求此3类LSA也必须在vlink穿越的transtit区域的LSDB中也存在，否则将不适用此3类LSA计算路由
  4. 存在vlink的ABR，通过vlink传来的3类LSA和vlink所经过的区域的其他ABR传来的3类LSA，在满足条件3的情况下，优先选择cost小的计算区域间的路由（这时候不一定区域0的3类LSA优于非骨干区域的3类LSA）

通过虚链路得到的3类LSA不会重传到所经过的区域

OSPF路由过滤

OSPF路由、LSA过滤

filter-policy（OSPF）

  1. filter-policy在ABR上如果对直连的区域1类LSA所产生的做过滤后，将不会产生此路由的3类LSA，ABR本地实现此路由过滤，并且此路由不会泛洪到其他区域。
  2. 在ABR上使用filter-policy对区域0的3类LSA所产生的路由也能实现路由的过滤，此路由不会出现在ABR的路由表中，并且ABR不会为直连的非骨干区域产生对应的此路由的3类LSA。
  3. filter-policy import方向 可以过滤任何类型的LSA所产生的路由，但不会过滤此路由的LSA（上面那两个是特殊）
  4. filter-policy export方向（设置对引入的路由在向外发布时进行过滤）仅能在ASBR上配置，因为5类LSA是由ASBR产生的
区域间LSA过滤

filter （OSPF区域）

此命令仅在ABR上配置。

filter export命令用来配置对区域内出方向的Type3 LSA进行过滤。（1、3类均可过滤）

filter import命令用来配置对区域内入方向的Type3 LSA进行过滤。（1、3类均可过滤）

LSA泛洪控制

ospf filter-lsa-out all在进程中仅对p2mp网络类型的邻居生效

ospf filter-lsa-out all在接口中优先在进程中

ospf filter-lsa-out all接口中不关心网络类型，都可以用。

OSPF外部路由汇总

ASBR汇总：

如果引入的明细路由cost type类型不一致，则默认为type2
如果引入的明细路由cost type类型一致，则继承明细路由的cost type

OSPF认证

认证模式分为3种：

  1. null 认证

type = 0 默认空认证

  1. 明文认证

type = 1 不能实现为伪造，篡改做判断

  1. md5认证

key id、key值需要一致

新增的字段：key id、data length（加密数据长度）、sequence Number（防重放攻击序列号）、data（哈希值）

type = 2 可防止报文篡改、伪造、重放攻击

哈希值=报文中的明文字段+key id+key值

认证实现：

一边做区域认证，一边接口认证是否可以建立邻居？

可以，只要认证类型相同，密码相同。

  1. 接口认证

接口认证优于区域认证

  1. 区域认证

vlink当作一个接口来做认证，vlink怎样都是属于区域0的。
