组播

组播的概述

  • 能够解决点到多点传输
  • 源发送一份数据，仅指定客户端接收该数据

单播特点：

  • 网路中的传输的信息量和需求该信息的用户成正比
  • 带宽消耗严重

广播特点：

  • 所有客户端均会收到数据包
  • 带宽浪费严重（浪费别人的带宽）

组播优点：

  • 降低网络流量、减轻服务器和CPU负载
  • 减少冗余流量、节约网络带宽、降低网络负载
  • 使多点应用成为可能

组播劣势：

  • 基于UDP
  • 尽力而为
  • 没有拥塞避免机制
  • 报文重复
  • 报文失序

组播的基本概念

组播组

  • 用IP组播地址标识的一组集合

区分不同的主播业务，接收组播流量的集合

组播源

  • 组播数据包的发送者

源是单播地址，目的地址是组播地址

组播成员

  • 所有加入某组播组的主机

组播路由器

  • 运行组播协议的设备

组播服务模型

依据接收者接收流量是否针对源进行区分，可分为：

  • ASM模型

  • SSM模型

ASM模型

Any-Source Multicast

ASM模型仅针对组地址提供组播分发。一个组播组地址作为一个网络服务的集合，任何源发布到该组地址的数据得到同样的服务。接收者主机加入组播组以后可以接收到任意源发送到该组的数据。

为了提高安全性，可以在路由器上配置针对组播源的过滤策略，允许或禁止来自某些组播源的报文通过。最终从接收者角度看，数据是经过筛选的。

ASM模型要求组地址必须整个组播网络中唯一。“唯一”指的是同一时刻一个ASM地址只能被一种组播应用使用。如果有两种不同的应用程序使用了同一个ASM组地址发送数据，它们的接收者会同时收到来自两个源的数据。这样一方面会导致网络流量拥塞，另一方面也会给接收者主机造成困扰。

客户端只能挑组播组不能挑组播源

SSM模型

Source-Specific Multicast

SSM模型针对特定源和组的绑定数据流提供服务，接收者主机在加入组播组时，可以指定只接收哪些源的数据或指定拒绝接收来自哪些源的数据。加入组播组以后，主机只会收到指定源发送到该组的数据。

SSM模型对组地址不再要求全网唯一，只需要每个组播源保持唯一。这里的“唯一”指的是同一个源上不同的组播应用必须使用不同的SSM地址来区分。不同的源之间可以使用相同的组地址，因为SSM模型中针对每一个（源，组）信息都会生成表项。这样一方面节省了组播组地址，另一方面也不会造成网络拥塞。

客户端不仅能对组播组作出选择还能对组播源做出选择

同一个组可提供不同的组播业务

组播协议

IGMP

  • 因特网组管理协议
  • 定义了一个网段内主机与组播路由器之间如何维护成员信息
  • 包括IGMPv1、IGMPv2、IGMPv3
IGMPv1

只支持ASM组播服务模型

IGMPv1的报文
  • 普遍组查询报文（General-Query）

  • 成员报告报文（Report）

字段 说明
Version IGMP版本，值为1.
Type 报文类型。
1.普遍组查询消息
2.成员报告消息
Group Address 组地址

普遍（通用）组查询报文（General-Query）

普遍组查询消息是查询器定期向共享网段内所有主机以主播方式（224.0.0.1）发送的查询消息，用于查询那些组播组存在的成员。

封装该消息的IP报文头的目的地址字段为224.0.0.1，本地网段上的所有路由器和主机都能识别并接收。

组地址字段全0，表示不指定组播组，默认情况下每60秒周期性发送，180秒为超时时间。

成员报告报文（Report）

成员报告消息是主机向主播路由器发送的报告消息，用于申请加入某个组播组或者应答查询消息。

封装该消息的IP报文头的目的地址字段为主机要加入的组播组地址，本网段上所有路由器及属于该组的所有主机都能够识别并接收，同一组的其他成员侦听到该报文后，不再发送针对相同组播组的报告，即报告被抑制。

组地址字段为主机要加入的组播组地址。

主机随机等待时间为10s

IGMPv1基本功能

IGMPv1协议主要基于查询和相应机制完成组播组管理。在连接用户主机的多路由器共享网段上，由三层路由协议选举出唯一的组播信息转发者（Assert Winner或DR），并作为IGMPv1的查询器，负责该网段的组成员关系查询。

因为IGMPv1不选举IGMP查询器，如果有多台路由器连接同一个多路访问网段时，就必须选举PIM Designated Router（DR）来避免重复查询，这个DR的选举，是由PIM进行的，选择IP地址高的为DR。PIM DR路由器在IGMPv1中充当IGMP查询器的功能，向主机发送IGMP查询。

IGMPv1查询器

IGMPv1选举查询器

  • 在pim-DM中通过比较hello消息上携带的优先级和IP地址，各路由器为多路由器网段选举指定路由器DR，充当IGMPv1的查询器

  • 当DR出现故障时，接收hello消息将会超时，邻居路由器之间会出发新的DR选举过程

  • IGMPv1的查询器失效时间就是DR的失效时间，其本质就是PIM邻居的失效时间，默认105s

  • IGMPv2和IGMPv3有自己的查询器选举机制和查询器失效时间。默认125秒

hello定时器
hello消息默认发送周期是30秒，超时时间默认为105秒

组成员关系维护（IGMPv1）

IGMP查询器周期性地发送General-Query消息，并启动定时器Timer-Membership，超时时间为180秒。发送周期可以使用命令配置，缺省值是60s。

网段内所有主机都接收到该General-Query消息，IGMPv1地主机都会发送report地消息

网段中的路由器接收到Report消息后，了解到本网段存在组播组G1和G2的成员，则由组播路由协议生成（*，G1）和（*，G2）组播转发项。网络中一旦有组播组G1和G2的数据到达路由器，将向该网段转发。

（*，G）：
*：任意源IP
G：主播组地址

新成员加入

host加入组播组，且侦听到网段中没有的组播数据，则不等待General-Query消息的到来，立即发送针对G3的Report消息。

网段中的路由器接收到Report消息后，了解到本网段内出现了组播组G3的成员，则在组播路由表项（*，G3）中添加对应的下游接口。网络中一旦有组播组G3的数据到达路由器，将向该网段中的路由器接收到Report消息后，了解到本网段内出现了组播组G3的成员，则在组播路由表项（*，G3）中添加对应的下游接口。网络中一旦有组播组G3的数据到达路由器，将向该网段转发。

组成员离开

退出组播组的主机将不再反馈Report消息。

由于该网段不存在其他的组成员，则路由器无法收到组的Report消息，定时器Timer-General-Query超时后（130秒），删除（*，G3）表项中对应的下游接口。如果过这时网络中组播组G3的数据到达路由器，则不会向该网段转发。

IGMPv2

只支持ASM组播服务模型

基本工作原理于IGMPv1基本相同。最大的不同在于IGMPv2增加离开组机制。成员主机离开组播时，会主动发送成员离开报文通知IGMP查询器；IGMP查询器收到成员离开报文后，会连续发送特定组查询报文，询问该组播组是否还存在组成员。如果过在一段时间内没有收到成员主机发送的报告报文，怎不在维护该组的组成员关系。

IGMPv2于IGMPv1区别

  • 增加了离组报文和特定组查询报文

  • 可控制（可以配置）主机的离开的最大响应时间，IGMPv1定死了，只能是10s

IGMPv2报文
  • 普遍组查询报文（General-Query）
    DIP 224.0.0.1 60s 周期性发送 180s内收不到组报告，认为成员全部离组。删除相应的（*，G）表项。
    最大响应时间：10s，可调可控.igmp max-reponse-time

  • 成员报告报文（Report）
    主机加组，主机响应通用组查询或是指定组查询发送。DIP：主机所属的组播组地址
    最大响应时间：0s，无实际意义。

  • 成员离开报文（Leave）
    目标地址为224.0.0.2（给所有路由器的）
    RFC2236建议：只有在将要离开的主机是发送了成员关系报告消息的最后一台主机时，才发送离组消息。
    因为如果过不是“最后一个响应者”的其他组成员静静的离开组播组，那么路由器也会继续转发组播会话，而不会发送查询消息，从而会一定程度上节省网络带宽。如果过这种行为不是必须的，那么也可以在任一一台主机离开组播时就发送离组消息。（思科路由器采用“最后一个响应者”发送leave消息，不是“最后一个响应者”在离开组时，不会发送leave消息）
    两种实现机制：主机离组就发送和“最后一个响应者”
    最大响应时间：0s，无实际意义。

  • 特定组查询报文（Group-Specific Query）
    DIP：离组报文的离组地址
    用于判断发送了离开报文的这个组是否还有成员
    发两次，以一秒为间隔。最大响应时间为1秒。如果两次都收不到应答则认为没有成员，随即删除（*，G）表项，不再向这个网段发送组播流
    可修改最大响应时间 igmp las tmember-quertinterval
    可修改发送次数 igmp robust-count

  • 版本1成员关系报告（Version 1 Membership report）
字段 说明
类型 0x11表示成员查询、0x12表示IGMPv1成员报告、0x16表示IGMPv2成员报告、0x17表示成员离开
最大响应时间 在发出响应报告前的以0.1秒为单位的最长时间，缺省值为10秒
校验和 对IGMP报文头部每16bit进行二进制反码求和
组地址 通用组成员查询=0.0.0.0、特定组成员查询=指定的组播组IP地址、成员报告=指定的组播组IP地址、离开报文=指定的组播组IP地址
IGMPv2查询器

查询器选举规则：

IGMPv2使用独立的查询器选举机制，当共享网段上存在多个组播路由器时，IP地址最小的路由器成为查询器。

  1. 最初，所有运行IGMPv2的组播路由器都认为自己是查询器，向本网段内的所有主机和组播路由器发送普遍组查询报文。在收到对方发送的普遍组查询报文后，将报文的源IP地址与自己的接口地址作比较。通过比较，IP地址最小的组播路由器将成为查询器，其他组播路由器成为非查询器（Non-Querier）。

  2. 此后，将由IGMP查询器向本网段内的所有主机和其他组播路由器发送普遍组查询报文，而非查询器（RouterB）则不再发送普遍组查询报文。

    非查询器上都会启动一个定时器（即其他查询器存在时间定时器Other Querier Present Timer）。在该定时器超时前，如果收到了来自查询器的查询报文，则重置该定时器；否则，就认为原查询器失效，并发起新的查询器选举过程。

IGMPv1查询器由PIM的DR或断言优胜者充当

IGMPv2查询器由接口地址小的路由器充当。可抢占。
非查询器可修改查询器超时时间。默认125s

其他IGMP查询器的存活时间 = 健壮系数（robust-count） * IGMP普遍组查询报文发送间隔 + （1/2）* 最大查询响应时间，以自身参数值进行计算。

也可以想改多少改多少：igmp timer other-querier-present优先于公式

IGMPv3

支持ASM、SSM组播服务模型

IGMPv3报文
  • 普遍组查询报文（General-Query）

  • 成员报告报文（Report）
    报告报文包括主机要加入地组和组播源地过滤模式
    INCLUDE
    EXCLUDE

  • 特定组查询报文（Group-Specific Query）

  • 特定源组查询报文（Group-and-Source-Specific Query）新增

  • 没有离组报文

  • 取消了成员关系报告抑制机制

IGMPv3报文分成两大类：查询报文和成员报告报文

IGMPv3没有定义专门的成员离开报文，成员离开通过特定类型的报告报文来传达。

IGMPv3中一个成员报告报文可以携带多个组播组信息，而之前的版本一个成员报告只能携带一个组播组。这样在IGMPv3中报文数量大大减少。

Record Type 组记录的类型。共分为三大类。
当前状态报告。用于对查询报文进行响应，通告自己目前的状态，共两种：

MODE_IS_INCLUDE，表示接收源地址列表包含的源发往该组的组播数据。如果指定源地址列表为空，该报文无效。

MODE_IS_EXCLUDE，表示不接收源地址列表包含的源发往该组的组播数据。
过滤模式改变报告。当组和源的关系在INCLUDE和EXCLUDE之间切换时，会通告过滤模式发生变化，共两种：

CHANGE_TO_INCLUDE_MODE，表示过滤模式由EXCLUDE转换到INCLUDE，接收源地址列表包含的新组播源发往该组播组的数据。如果指定源地址列表为空，主机将离开组播组。

CHANGE_TO_EXCLUDE_MODE，表示过滤模式由INCLUDE转换到EXCLUDE，拒绝源地址列表包含的新组播源发往该组的组播数据。
源列表改变报告。当指定源发生改变时，会通告源列表发生变化，共两种：

ALLOW_NEW_SOURCES，表示在现有的基础上，需要接收源地址列表包含的组播源发往该组播组的组播数据。如果当前对应关系为INCLUDE，则向现有源列表中添加这些组播源；如果当前对应关系为EXCLUDE，则从现有阻塞源列表中删除这些组播源。

BLOCK_OLD_SOURCES，表示在现有的基础上，不再接收源地址列表包含的组播源发往该组播组的组播数据。如果当前对应关系为INCLUDE，则从现有源列表中删除这些组播源；如果当前对应关系为EXCLUDE，则向现有源列表中添加这些组播源。
Aux Data Len 辅助数据长度。在IGMPv3的报告报文中，不存在辅助数据字段，该字段设为0。
Number of Sources 本记录中包含的源地址数量。
Multicast Address 组播组地址。
Sources Address 组播源地址。
Auxiliary Data 辅助数据。预留给IGMP后续扩展或后续版本。在IGMPv3的报告报文中，不存在辅助数据。
报告报文组记录

IGMPv3报告报文组记录

  • 通过报告报文组记录字段记录组播组与组播源列表对应关系发生的变化

IGMP不同版本互操作

与IGMPv1路由器兼容

  • IGMPv2主机必须发送版本1成员报告报文，且不发送成员离开报文
  • IGMPv2路由器必须强制发送版本1报文

与IGMPv1主机兼容

  • IGMPv2主机成员报告报文需被版本1主机抑制
  • IGMPv2路由器忽略收到的成员离开报文

IGMP SSM Mapping
  • 为IGMPv1主机和IGMPv2主机提供SSM服务
  • 通过再路由器上静态配置SSM地址的映射规则，将IGMPv1和IGMPv2报文报告中的（*，G）信息转化为对应的（S，G）信息

配置了SSM Mapping规则后，当IGMP查询器收到来自成员主机的IGMPv1或IGMPv2报告报文时，首先检查该报文中所携带的组播组地址G，然后根据检查结果的不同分别进行处理。

注：IGMP SSM Mapping不处理IGMPv3的报告报文。为了保证同一网段运行任意版本IGMP的主机都能得到SSM服务，需要在与成员主机所在网段相连的组播路由器接口上运行IGMPv3。
